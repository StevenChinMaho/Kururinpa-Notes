# 3.5 Symbolic Constants

## 一、Symbol vs. Variable 比較

||Symbol（符號常數）|Variable（變數）|
|---|---|---|
|使用資料記憶體？|**No**|Yes|
|執行時可改變？|**No**|Yes|

> 符號常數在**組譯時期**就已決定值，不佔用資料記憶體空間。

---

## 二、Equal-Sign 指令（`=`）

### 語法

```asm
name = expression
```

### 特性

- expression 必須是 **32-bit 整數**或常數運算式
- **可以重複定義**（redefined）
- name 稱為「symbolic constant」

### 範例

```asm
COUNT = 500

mov ax, COUNT   ; 等同於 mov ax, 500
```

> ⚠️ `mov COUNT, 500` 是**錯誤**的，COUNT 不是記憶體位置，是 immediate value。

### 重複定義範例

```asm
COUNT = 100
; ... 程式碼 ...
COUNT = 200   ; 合法，可重新定義
```

## 三、計算陣列大小（`$` 運算子）

`$` 代表**目前位置計數器（current location counter）**，可搭配陣列起始位址計算元素數量。

### Byte 陣列

```asm
list     BYTE 10, 20, 30, 40
ListSize = ($ - list)        ; = 4（bytes）
```

### Word 陣列

```asm
list     WORD 1000h, 2000h, 3000h, 4000h
ListSize = ($ - list) / 2   ; = 4（每個 WORD 佔 2 bytes）
```

### Doubleword 陣列

```asm
list     DWORD 1, 2, 3, 4
ListSize = ($ - list) / 4   ; = 4（每個 DWORD 佔 4 bytes）
```

> 規則：`($ - 陣列起始名稱) / 每個元素的 bytes 數`

## 四、EQU 指令

### 語法

```asm
name EQU expression
name EQU <text>
```

### 特性

- 可定義**整數**或**文字**運算式
- **不可重複定義**（不同於 `=`）
- 文字類型必須用 `< >` 包圍

### 範例

```asm
PI        EQU <3.1416>
pressKey  EQU <"Press any key to continue...", 0>

.data
prompt BYTE pressKey    ; 展開為字串
```

> ⚠️ `PI = 3.1416` 是**錯誤**的，因為 `=` 只接受整數，非整數須用 EQU。

## 五、TEXTEQU 指令

### 語法

```asm
name TEXTEQU <text>
name TEXTEQU %(expression)   ; % 表示求運算式的值
name TEXTEQU anotherTextEquName
```

### 特性

- 定義**文字巨集（text macro）**
- 可定義整數或文字運算式
- **可以重複定義**（不同於 EQU）
- `%` 前置符號：強制**求出運算式數值**再存入

### 範例

```asm
continueMsg TEXTEQU <"Do you wish to continue (Y/N)?>

rowSize = 5
count   TEXTEQU %(rowSize * 2)  ; count = "10"

setupAL TEXTEQU <mov al, count> ; 巨集文字

.data
prompt1 BYTE continueMsg

.code
setupAL     ; 展開為 → mov al, 10
```

## 六、三種符號常數比較

|指令|可否重定義|接受型態|說明|
|---|---|---|---|
|`=`|✅ 可以|整數only|最簡單的符號常數|
|`EQU`|❌ 不可|整數 / 文字|適合定義不變的常數或字串|
|`TEXTEQU`|✅ 可以|整數 / 文字|文字巨集，功能最靈活|

---

## 七、重點整理

```
符號常數不佔記憶體，值在組譯時期決定
= 只接受整數，可重複定義
$ 代表目前位置，用來計算陣列大小
EQU 接受整數與文字，但不可重複定義
TEXTEQU 為文字巨集，% 可求運算式的值
陣列大小公式：($ - 陣列名稱) / sizeof(元素)
```
