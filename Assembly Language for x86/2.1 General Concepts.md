### 1. 微電腦基本設計 (Basic Microcomputer Design)

微電腦的核心由中央處理器 (CPU) 與多種匯流排構成：

- **CPU 主要元件**：
    
    - **控制單元 (Control Unit, CU)**：負責協調各項執行步驟的順序 。
        
    - **算術邏輯單元 (ALU)**：負責執行算術與位元運算 。
        
    - **時脈 (Clock)**：用來同步化 CPU 的所有操作 。
        
    - **暫存器 (Registers)**：CPU 內部的高速記憶單元 。
	
	![[2.1.1.png]]

- **匯流排 (Buses)**：
    
    - **資料匯流排 (Data bus)**：在 CPU 與記憶體之間傳遞指令與資料 。
        
    - **輸出入匯流排 (I/O bus)**：在 CPU 與系統輸出入裝置間傳遞資料 。
        
    - **控制匯流排 (Control bus)**：使用二進位訊號來同步連接到系統匯流排上所有裝置的動作 。
        
    - **位址匯流排 (Address bus)**：當正在執行的指令要傳遞資料時，用來保存指令與資料的位址 。

### 2. 時脈 (Clock) 與等待狀態 (Wait States)

- **機器週期 (Machine Cycle)**：用來測量單一操作所需的時間 。
    
- 一個機器指令至少需要一個時脈週期來執行，部分指令甚至需要超過 50 個週期 。
    
- **等待狀態 (Wait states)**：由於 CPU、系統匯流排與記憶體電路之間的運行速度存在差異 ，有時會產生空白的時脈週期，稱為等待狀態 。

### 3. 指令執行週期 (Instruction Execution Cycle)

CPU 執行一道指令通常包含以下五個基本階段：

1. **Fetch (擷取)**：從快取或記憶體中取得指令 。
    
2. **Decode (解碼)**：分析指令內容 。
    
3. **Fetch operands (擷取運算元)**：如果指令需要，則取得輸入的資料 。
    
4. **Execute (執行)**：利用 ALU 或浮點運算單元執行該指令 。
    
5. **Store output (儲存結果)**：將運算結果寫回暫存器或記憶體 。

### 4. 記憶體讀取週期 (Reading from Memory)

從記憶體讀取資料通常需要多個機器週期，因為記憶體的反應速度遠慢於 CPU 內部的暫存器 。讀取單一數值的過程分為四個步驟：

- **週期 1**：將位址放置於位址匯流排 (Address bus) 上 。
    
- **週期 2**：將讀取線 (Read Line, RD) 設為低電位 。
    
- **週期 3**：CPU 等待一個週期，讓記憶體有時間反應 。
    
- **週期 4**：讀取線 (RD) 轉為高電位 (1)，表示資料已經準備好並放置在資料匯流排上 。

### 5. 快取記憶體 (Cache Memory)

快取是一種昂貴但高速的靜態隨機存取記憶體 (SRAM) ：

- **Level-1 (L1) cache**：建置在 CPU 內部 。
    
- **Level-2 (L2) cache**：建置在 CPU 外部 。
    
- **快取命中 (Cache hit)**：要讀取的資料已經存在快取記憶體中 。
    
- **快取未命中 (Cache miss)**：要讀取的資料不在快取記憶體中 。
    

### 6. 程式的執行流程 (How a Program Runs)

當使用者啟動程式時，作業系統會執行以下流程：

1. 使用者送出程式名稱 。
    
2. 作業系統在目錄中尋找該程式 。
    
3. 從目錄區塊中取得程式的起始叢集 (starting cluster) 。
    
4. 將程式載入記憶體並開始執行 。
