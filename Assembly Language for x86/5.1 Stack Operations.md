# Stack Operations（堆疊操作）

> Assembly Language for x86 Processors (7th Edition) — Chapter 5: Procedures

## 本節涵蓋主題

| 主題 | 說明 |
|---|---|
| Runtime Stack | CPU 管理的 LIFO 記憶體結構，由 ESP 指向頂端 |
| PUSH | 將值壓入堆疊，ESP 減少 |

| PUSHFD / POPFD | 保存 / 還原 EFLAGS 旗標暫存器 |

---

## 1. Runtime Stack（32-Bit 模式）

Runtime Stack 是由 CPU 直接管理的 LIFO（Last-In, First-Out）記憶體區段。

- 使用 **ESP**（Extended Stack Pointer）暫存器追蹤頂端位置。
- ESP **始終指向最後一個被 PUSH 進來的值**。
- 堆疊的成長方向：**由高記憶體位址往低記憶體位址**。

```
記憶體位址示意（高 → 低）：

Offset      Value
00001000  [ 00000006 ]  ← ESP（目前頂端）
00000FFC  [          ]
00000FF8  [          ]
00000FF4  [          ]
00000FF0  [          ]  ← lower（尚未使用）
```

> **關鍵觀念**：堆疊頂端在低位址方向，每次 PUSH 讓 ESP 往下移；每次 POP 讓 ESP 往上移。

---

## 2. PUSH 操作

32-bit PUSH 分兩步執行：

1. ESP = ESP − 4（騰出空間）
2. 將值複製到 ESP 所指的位置

16-bit PUSH 則是 ESP = ESP − 2。

**PUSH 語法**：

```asm
PUSH r/m16    ; 推入 16-bit 暫存器或記憶體
PUSH r/m32    ; 推入 32-bit 暫存器或記憶體
PUSH imm32    ; 推入 32-bit 立即數
```

**範例：連續 PUSH 三個值**

```asm
; 初始 ESP = 00001000h，值為 00000006h
push eax     ; 假設 EAX = 000000A5h → ESP = 00000FFCh
push ebx     ; EBX = 00000001h      → ESP = 00000FF8h
push ecx     ; ECX = 00000002h      → ESP = 00000FF4h
```

結果：

```
Offset      Value
00001000  [ 00000006 ]
00000FFC  [ 000000A5 ]
00000FF8  [ 00000001 ]
00000FF4  [ 00000002 ]  ← ESP
```

> ESP 下方的空間（低位址方向）隨時可用，除非堆疊已溢位（stack overflow）。

---

## 3. POP 操作

POP 分兩步執行：

1. 將 ESP 所指位置的值複製到目標運算元（暫存器或記憶體變數）
2. ESP = ESP + n（n 為 2 或 4，依目標運算元大小決定）

**POP 語法**：

```asm
POP r/m16    ; 取出 16-bit 值
POP r/m32    ; 取出 32-bit 值
```

> 注意：POP 後，舊位置的值仍殘留在記憶體中，但 ESP 已移過去，邏輯上視為「不存在」。

## 4. 相關批次指令

| 指令 | 說明 |
|---|---|
| `PUSHFD` | 將 EFLAGS（32-bit）壓入堆疊 |
| `POPFD` | 從堆疊取出值並載入 EFLAGS |
| `PUSHAD` | 依序 PUSH 所有 32-bit 通用暫存器：EAX、ECX、EDX、EBX、ESP、EBP、ESI、EDI |
| `POPAD` | 以相反順序 POP 所有 32-bit 通用暫存器 |
| `PUSHA` | 同 PUSHAD，但針對 16-bit 暫存器 |
| `POPA` | 同 POPAD，但針對 16-bit 暫存器 |

## 5. Stack 的應用場景

| 應用 | 說明 |
|---|---|
| 暫存器暫存 | 在使用暫存器前先 PUSH 保存原值，使用後 POP 還原 |
| Return Address | 執行 `CALL` 時，CPU 自動將返回位址 PUSH 到堆疊 |
| 引數傳遞 | 呼叫 Procedure 時透過堆疊或暫存器傳入輸入值 |
| 區域變數 | Procedure 內部的暫時變數儲存於堆疊 |

## 6. 使用 PUSH / POP 保存暫存器

**重要原則**：PUSH 與 POP 的順序必須**相反**（先進後出）。

```asm
push esi          ; 保存原始值（因為下面要覆寫）
push ecx
push ebx

mov esi, OFFSET dwordVal   ; 設定記憶體區塊起始位址
mov ecx, LENGTHOF dwordVal ; 設定元素個數
mov ebx, TYPE dwordVal     ; 設定每個元素大小
call DumpMem               ; 呼叫 Library 函式

pop ebx           ; 還原（與 PUSH 相反順序）
pop ecx
pop esi
```

## 7. 範例：巢狀迴圈（Nested Loop）

ECX 被 LOOP 指令隱含使用，巢狀迴圈時需用堆疊保存外層計數器：

```asm
mov ecx, 100    ; 外層迴圈計數

L1:
    push ecx        ; 保存外層 ECX
    mov ecx, 20     ; 內層迴圈計數

L2:
    ; 內層迴圈主體
    loop L2         ; ECX-- ≠ 0 則跳 L2

    pop ecx         ; 還原外層 ECX
    loop L1         ; 繼續外層迴圈
```

## 8. 範例：字串反轉（Reversing a String）

利用堆疊的 LIFO 特性反轉字串。

```asm
; aName BYTE "Abraham Lincoln", 0
; nameSize = ($ - aName) - 1

; === Phase 1：將每個字元 PUSH 到堆疊 ===
mov ecx, nameSize
mov esi, 0
L1:
    movzx eax, aName[esi]  ; 取出 1 個字元（零擴展到 32-bit）
    push eax                ; PUSH（只能 PUSH word/dword）
    inc esi
    loop L1

; === Phase 2：從堆疊 POP 回陣列（順序已反轉）===
mov ecx, nameSize
mov esi, 0
L2:
    pop eax                 ; 取出（LIFO → 反轉順序）
    mov aName[esi], al      ; 只取低 8-bit 存回
    inc esi
    loop L2
```

> **問：為何字元要先放入 EAX 再 PUSH？**  
> 因為堆疊只支援 16-bit 或 32-bit 的推入操作，無法直接 PUSH 一個 BYTE，必須先零擴展到 32-bit。

## 重點整理

| 指令 / 概念 | 功能 | 關鍵注意事項 |
|---|---|---|
| `PUSH r/m32` | ESP−4，值存入 [ESP] | 堆疊向低位址成長 |
| `POP r/m32` | 值從 [ESP] 取出，ESP+4 | 舊值仍殘留於記憶體，但邏輯上無效 |
| `PUSHFD` / `POPFD` | 保存 / 還原 EFLAGS | 常用於保護旗標不被子程序污染 |
| `PUSHAD` / `POPAD` | 一次保存 / 還原全部通用暫存器 | POPAD 以反向順序還原 |
| PUSH/POP 對稱原則 | PUSH 順序與 POP 順序相反 | 順序錯誤會造成暫存器值錯亂 |
| Nested Loop | 進入內層前 PUSH ECX，離開後 POP ECX | LOOP 指令隱含操作 ECX |
| 字串反轉 | 利用 LIFO 特性，PUSH 後 POP 自然反轉 | BYTE 需先 MOVZX 到 EAX 才能 PUSH |
