### 1. 輸出入的四個層級 (Levels of Input-Output)

電腦系統在執行 I/O 操作時，由上到下可劃分為不同的抽象層級：

- **Level 3：高階語言函式 (High-level language function)**
    
    - **範例**：C++、Java 等語言所提供的標準函式庫 。
        
    - **特性**：具有良好的跨平台可攜性 (portable) 且方便開發，但執行速度不一定是絕對最快的 。
        
- **Level 2：作業系統 (Operating system)**
    
    - **範例**：應用程式介面 (API) 。
        
    - **特性**：提供了作業系統延伸的強大功能，但對於開發者而言，有許多繁雜的技術細節需要去掌握 。
        
- **Level 1：BIOS**
    
    - **特性**：包含直接與硬體裝置通訊的底層驅動程式 。
        
    - **限制**：現代作業系統的安全性機制 (OS security)，可能會阻止一般的應用程式直接在這個層級進行操作 。
        
- **Level 0：硬體 (Hardware)**
    
    - 最底層的實體設備本身。
        

### 2. 實際運作流程：顯示一段字串

當一個高階語言程式要在螢幕上顯示一段字串時，指令的傳遞會依序向下穿透這些層級 ：

1. **Level 3**：應用程式 (Application Program) 發出顯示要求 。
    
2. **Level 2**：呼叫作業系統函式 (OS Function) 處理 。
    
3. **Level 1**：呼叫 BIOS 函式 (BIOS Function) 轉譯為硬體指令 。
    
4. **Level 0**：最終由硬體 (Hardware) 執行動作並顯示在螢幕上 。
    

### 3. 組合語言的特權 (Programming levels for ASM)

相較於高階語言通常被限制在 Level 3 往下呼叫，**組合語言 (Assembly language) 程式具備極大的彈性** 。 寫組合語言的程式設計師可以根據需求，直接在以下**任何一個層級**執行輸出入操作：

- 直接呼叫高階函式庫 (Library - Level 3) 。
    
- 直接呼叫作業系統函式 (OS Function - Level 2) 。
    
- 直接呼叫 BIOS 函式 (BIOS Function - Level 1) 。
    
- 甚至是直接對硬體 (Hardware - Level 0) 進行控制 。
