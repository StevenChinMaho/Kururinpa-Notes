## 1. 建立程序（Creating Procedures）

大型問題可拆分成小任務，每個任務寫成一個 **Procedure**（程序）。  
Procedure 相當於 C++ / Java 中的函式（function / method）。

**宣告語法**：使用 `PROC` 與 `ENDP` Directive，且程序必須有名稱。

```asm
; 啟動程序（程式進入點，不需要 RET）
main PROC
    ...
main ENDP

; 一般程序（必須以 RET 結尾）
sample PROC
    ...
    ret
sample ENDP
```

> 除了 `main`（程式啟動程序）之外，**所有其他程序都必須以 `RET` 結尾**，否則執行流程不會返回。

## 2. 程序內的 Label（Labels in Procedures）

### 區域 Label（Local Label）

預設情況下，Label 只在**所屬程序內**可見，`JMP` 指令無法跳到其他程序的 Label。

```asm
main PROC
    jmp L2      ; 錯誤：L2 在 sub2 裡，main 看不到
    ...
main ENDP

sub2 PROC
L2:             ; 區域 Label，只在 sub2 內有效
    jmp L1      ; 正確：L1 是全域 Label
    ret
sub2 ENDP
```

### 全域 Label（Global Label）

使用**雙冒號** `::` 宣告全域 Label，整個程式皆可見。

```asm
L1::            ; 全域 Label，任何程序都能 JMP 到這裡
```

> **警告**：跨程序的 JMP 會破壞 Stack 上的 Return Address，通常是不好的設計，應避免。

## 3. CALL 與 RET 指令

### CALL

`CALL` 執行時做兩件事：

1. 將**下一條指令的位址**（Return Address）PUSH 到 Stack
2. 將被呼叫程序的起始位址載入 **EIP**（Instruction Pointer）

### RET

`RET` 執行時：

- 從 Stack 頂端 POP 出 Return Address，載入 EIP
- 執行流程回到 `CALL` 後的下一條指令

### 圖解

```
main PROC
    00000020  call MySub     ← CALL 執行：PUSH 00000025, EIP = 00000040
    00000025  mov eax, ebx   ← RET 後回到這裡
    ...
main ENDP

MySub PROC
    00000040  mov eax, edx   ← CALL 後從這裡開始執行
    ...
    ret                      ← POP 00000025 → EIP，回到 main
MySub ENDP
```

**Stack 狀態（CALL 執行後、RET 執行前）**：

```
[ 00000025 ]  ← ESP（Return Address）
[           ]
```

**RET 執行後**：

```
[ 00000025 ]  （ESP 往上移，邏輯上已移除）
              ESP → 指向更高位址
EIP = 00000025
```

## 4. 巢狀程序呼叫（Nested Procedure Calls）

每次 `CALL` 都會在 Stack 上壓一個 Return Address，多層呼叫時 Stack 會依序堆積。

```asm
main  → call Sub1  → call Sub2  → call Sub3
```

當 Sub3 執行時，Stack 上有三層 Return Address：

```
[ ret to main  ]  ← 最早 PUSH（main 裡的 exit 位址）
[ ret to Sub1  ]  ← Sub1 裡 ret 後的位址
[ ret to Sub2  ]  ← ESP（最新，Sub2 裡 ret 後的位址）
```

各 `RET` 依序 POP，執行流程正確地一層層返回：  
Sub3 → Sub2 → Sub1 → main。

## 5. 程序文件慣例（Documenting Procedures）

為提高可讀性與可重用性，建議在每個程序前加上標準化的注解區塊：

| 欄位 | 說明 |
|---|---|
| **描述** | 程序的功能說明 |
| **Receives** | 輸入參數清單，說明用途與要求 |
| **Returns** | 回傳值說明（通常放在哪個暫存器）|
| **Requires** | 前置條件（Preconditions），呼叫前必須滿足的條件 |

> 若前置條件未被滿足就呼叫程序，通常不會產生預期的輸出。

**範例：SumOf 程序**

```asm
;-----------------------------------------------------
SumOf PROC
;
; 計算三個 32-bit 整數的總和並回傳。
; Receives: EAX、EBX、ECX — 三個整數，可為有號或無號。
; Returns:  EAX = 三數之和；相關 Status Flags（Carry、
;           Overflow 等）會被修改。
; Requires: nothing
;-----------------------------------------------------
    add eax, ebx
    add eax, ecx
    ret
SumOf ENDP
```

## 6. 透過暫存器傳遞引數（Passing Register Arguments）

在組合語言中，最常見的引數傳遞方式是**直接把值放入通用暫存器**，再呼叫程序。

```asm
.data
theSum DWORD ?

.code
main PROC
    mov eax, 10000h    ; 引數一
    mov ebx, 20000h    ; 引數二
    mov ecx, 30000h    ; 引數三
    call SumOf         ; EAX = EAX + EBX + ECX
    mov theSum, eax    ; 儲存結果
main ENDP
```

## 7. 範例：ArraySum 程序

接收陣列指標與元素個數，回傳總和於 EAX。

```asm
ArraySum PROC
; Receives: ESI = 陣列（DWORD）的起始位址
;           ECX = 元素個數
; Returns:  EAX = 所有元素的總和
;-----------------------------------------------------
    push esi           ; 保存 ESI、ECX（程序內部會修改）
    push ecx
    mov eax, 0         ; 總和歸零

L1:
    add eax, [esi]         ; 累加目前元素
    add esi, TYPE DWORD    ; ESI 移到下一個元素（+4）
    loop L1                ; ECX-- ≠ 0 則繼續

    pop ecx            ; 還原暫存器（與 PUSH 相反順序）
    pop esi
    ret                ; EAX = 總和
ArraySum ENDP
```

## 8. 保存與還原暫存器（Saving and Restoring Registers）

### 原則

- **被程序修改的暫存器**，進入時 PUSH、離開前 POP。
- **唯一例外**：作為回傳值的暫存器（通常是 EAX），**不應** PUSH/POP，否則回傳值會被覆蓋。

### 錯誤示範：不應 PUSH 回傳值暫存器

```asm
SumOf PROC
    push eax       ; 錯誤！將原本的 EAX 先存入 Stack
    add eax, ebx   ; EAX = EAX + EBX + ECX
    add eax, ecx
    pop eax        ; 錯誤！把舊值蓋回 EAX，總和遺失！
    ret
SumOf ENDP
```

> 總和在 `add` 後已存入 EAX，但 `pop eax` 把舊值蓋了回去，導致**回傳值遺失**。

## 9. USES 運算子

`USES` 是 MASM 提供的便利語法，可在宣告程序時列出要保護的暫存器，由組譯器自動產生對應的 PUSH / POP 程式碼。

```asm
ArraySum PROC USES esi ecx
    mov eax, 0
    ...
    ret
ArraySum ENDP
```

MASM 自動展開為：

```asm
ArraySum PROC
    push esi       ; ← 自動產生
    push ecx       ; ← 自動產生
    mov eax, 0
    ...
    pop ecx        ; ← 自動產生（反向）
    pop esi        ; ← 自動產生（反向）
    ret
ArraySum ENDP
```

> `USES` 讓程式碼更簡潔，且不易犯 PUSH/POP 順序錯誤。

## 重點整理

| 指令 / 概念 | 功能 | 關鍵注意事項 |
|---|---|---|
| `PROC` / `ENDP` | 宣告程序的起點與終點 | 程序必須有名稱 |
| `RET` | 從 Stack POP Return Address → EIP | 除 `main` 外，所有程序都必須有 `RET` |
| `CALL` | PUSH Return Address，跳入程序 | EIP = 被呼叫程序的起始位址 |
| 區域 Label（`L1:`） | 只在所屬程序內可見 | 不可用 JMP 跨程序跳轉 |
| 全域 Label（`L1::`） | 整個程式皆可見 | 雙冒號宣告；跨程序 JMP 仍不建議 |
| 巢狀呼叫 | 每層 CALL 壓一個 Return Address | Stack 依序堆疊，RET 依序還原 |
| 引數傳遞 | 呼叫前將值放入指定暫存器 | 是 x86 組合語言最常見的傳參方式 |
| 保護暫存器 | 進入時 PUSH，離開前 POP | 回傳值暫存器（EAX）不應 PUSH/POP |
| `USES` | 自動產生 PUSH / POP，保護指定暫存器 | 由 MASM 組譯器展開，順序自動正確 |
