# 3.1 Basic Language Elements

## 1. 第一個組合語言程式與變數宣告 (First Assembly Language Program)

- **基本程式架構**：使用 `main PROC` 與 `main ENDP` 來標示主程式的範圍 。
    
- **基本指令**：例如 `mov eax,5`（將 5 移入 eax 暫存器）與 `add eax,6`（將 eax 暫存器的值加 6）。
    
- **結束程式**：使用 `INVOKE ExitProcess,0` 呼叫 Windows 服務來結束程式 。
    
- **加入變數**：
    
    - 使用 `.data` 區段來宣告變數，例如 `Sum DWORD 0`（建立一個名為 Sum 的變數）。
        
    - 使用 `.code` 區段來撰寫程式碼 。
        

## 2. 整數常值 (Integer Literals)

- **語法格式**：`[{+|-}] digits [radix]`（括號表示為可選項）。
    
- **基底字元 (Radix characters)**：
    
    - `h` - 十六進位 (hexadecimal) 。
        
    - `d` - 十進位 (decimal) 。
        
    - `b` - 二進位 (binary) 。
        
    - `o` - 八進位 (octal) 。
        
- **重要規則**：如果十六進位常值以「字母」開頭（例如 A~F），**必須在最前面加上 `0`**（例如 `0A5h`），以防止組譯器將其誤認為是識別字 (identifier) 。
    

## 3. 常數整數運算式 (Constant Integer Expressions)

運算式在組譯階段就會計算完畢。包含以下運算子與優先層級：

|**運算子**|**名稱**|**優先層級**|
|---|---|---|
|`()`|括號 (parentheses)|1|
|`+`, `-`|一元正負號 (unary plus, minus)|2|
|`*`, `/`|乘、除 (multiply, divide)|3|
|`MOD`|取餘數 (modulus)|3|
|`+`, `-`|加、減 (add, subtract)|4|

## 4. 實數常值 (Real Number Literals)

- **語法格式**：`[sign]integer.[integer][exponent]` 。
    
- **指數格式**：`E[{+,-}]integer` 。
    
- **範例**：`2.`、`+3.0`、`-44.2E+05`（等於 -44.2 x 10^5）。
    

## 5. 字元與字串常值 (Character and String Literals)

- **字元常值**：使用單引號或雙引號包覆（如 `'A'`, `"d"`）。一個 ASCII 字元佔用 1 byte，例如 `'A'` 在記憶體中儲存為數字 65（或 16 進位的 41h）。
    
- **字串常值**：同樣使用單引號或雙引號包覆（如 `"ABC"`, `'xyz'`），每個字元佔用 1 byte 。支援內嵌引號，例如：`'Say "Goodnight," Gracie'` 。
    

## 6. 保留字與識別字 (Reserved Words and Identifiers)

- **保留字 (Reserved Words)**：不能做為識別字使用 。包含指令助憶碼（如 MOV, ADD）、假指令 (directives)、屬性、運算子與預設符號（如 @data）。
    
- **識別字 (Identifiers)**：用於變數名稱、程序名稱或標籤 。
    
    - 長度為 1-247 個字元，可包含數字 。
        
    - **不在乎大小寫 (not case sensitive)** 。
        
    - 第一個字元必須是字母、`_`、`@`、`?` 或 `$` 。
        

## 7. 假指令 (Directives) 與 指令 (Instructions)

- **假指令 (Directives)**：
    
    - 由「組譯器 (assembler)」辨識與執行的命令，**不屬於** Intel 指令集的一部分 。
        
    - 用於宣告程式碼、資料區段、選擇記憶體模式、宣告程序等（例如：`.data`, `.code`, `.stack 100h`）。
        
    - 不區分大小寫 。不同的組譯器有不同的假指令（如 NASM 與 MASM 不同）。
        
- **指令 (Instructions)**：
    
    - 由 CPU 在執行時期 (runtime) 執行 。
        
    - 由組譯器翻譯成機器碼 (machine code) 。
        
    - 格式為：`[label:] mnemonic [operands][;comment]` 。
        

## 8. 標籤 (Labels)

作為程式碼或資料的位址標記 (place markers) 。必須遵守識別字的命名規則 。

- **資料標籤 (Data label)**：名稱必須是唯一的（例如 `count DWORD 100`）。
    
- **程式碼標籤 (Code label)**：作為跳躍 (jump) 與迴圈 (loop) 指令的目標，後方必須加上冒號（例如 `L1:`）。
    

## 9. 助憶碼 (Mnemonics) 與 運算元 (Operands)

- **助憶碼**：用於輔助記憶的指令名稱，如 MOV, ADD, SUB 等 。
    
- **運算元**：每個指令可以有 0 到 3 個運算元 。
    
    - 包含：常數 (Constant)、常數運算式、暫存器 (Register)、記憶體 (memory / data label) 。
        
    - 常數或常數運算式通常被稱為「立即值 (immediate values)」。
        

## 10. 註解 (Comments)

- **單行註解**：以分號 `;` 開頭 。
    
- **多行註解**：使用 `COMMENT` 假指令加上一個程式設計師自訂的符號來包覆（例如 `COMMENT @ 內容 @`）。
    

## 11. NOP 指令 (No Operation)

- 不執行任何操作，單純佔用 1 byte 的程式空間 。
    
- 主要用途是將程式碼對齊到有效的記憶體邊界（例如雙字組偶數位址），讓 x86 CPU 能更快速地載入程式碼與資料 。
    

## 12. 指令格式範例 (Instruction Format Examples)

- **零個運算元**：`stc` (設定 Carry flag) 。
    
- **一個運算元**：`inc eax` (暫存器加 1)、`inc myByte` (記憶體加 1) 。
    
- **兩個運算元**：`add ebx,ecx` (暫存器, 暫存器)、`sub myByte,25` (記憶體, 常數)、`add eax, 36*25` (暫存器, 常數運算式) 。
