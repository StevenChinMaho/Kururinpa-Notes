## 本節涵蓋主題

|運算子 / 指令|功能簡述|
|---|---|
|`OFFSET`|取得標籤相對資料段起始位址的偏移量（位元組數）|
|`ALIGN`|對齊變數到特定邊界|
|`PTR`|覆寫運算元的宣告大小|
|`TYPE`|取得單一元素的大小（位元組）|
|`LENGTHOF`|取得陣列元素的數量|
|`SIZEOF`|取得整個陣列佔用的位元組數（= LENGTHOF × TYPE）|
|`LABEL`|為既有記憶體位置指定另一個名稱與型別|

---

## 1. OFFSET 運算子

`OFFSET` 回傳標籤（label）距資料段起始位址的**位元組偏移量**。

- **保護模式**（Protected mode）：回傳 32 位元（雙字組）
- **實模式**（Real mode）：回傳 16 位元

### 範例

```asm
.data
bVal  BYTE  ?        ; 假設位於 00404000h
wVal  WORD  ?        ; 00404001h
dVal  DWORD ?        ; 00404003h
dVal2 DWORD ?        ; 00404007h

.code
mov esi, OFFSET bVal   ; ESI = 00404000h
mov esi, OFFSET wVal   ; ESI = 00404001h
mov esi, OFFSET dVal   ; ESI = 00404003h
mov esi, OFFSET dVal2  ; ESI = 00404007h
```

### 搭配陣列偏移

```asm
.data
myArray WORD 1,2,3,4,5

.code
mov esi, OFFSET myArray + 4  ; 指向第三個元素（每個 WORD = 2 bytes）
mov eax, [esi]               ; EAX = 00040003h（間接定址）
```

### 對應 C/C++ 概念

```cpp
// C++
char array[1000];
char *p = array;     // p 儲存 array 的起始位址
```

```asm
; Assembly
.data
array BYTE 1000 DUP(?)
.code
mov esi, OFFSET array    ; esi 儲存 array 的起始位址
```

## 2. ALIGN 指令

將變數對齊到指定的邊界，`bound` 可為 **1、2、4、8、16**。

```asm
ALIGN bound
```

### 範例

```asm
bVal  BYTE  ?   ; 00404000h
ALIGN 2         ; 補到 2 的倍數 → 跳到 00404002h
wVal  WORD  ?   ; 00404002h
bVal2 BYTE  ?   ; 00404004h
ALIGN 4         ; 補到 4 的倍數 → 跳到 00404008h
dVal  DWORD ?   ; 00404008h
dVal2 DWORD ?   ; 0040400Ch
```

> **注意**：若目前位址本身已符合對齊條件（如 DWORD 後接 `ALIGN 4`），則不做任何動作。

## 3. PTR 運算子

**覆寫**運算元原本宣告的大小，讓你可以存取變數的部分位元組。

### 基本語法

```asm
TYPE PTR variable
```

### 小端序（Little Endian）補充

Intel 以**小端序**儲存多位元組整數，最低有效位元組存在最低位址。

```
DWORD 12345678h 在記憶體中的排列：
位址  值
0000  78   ← 最低位址存最低位元組
0001  56
0002  34
0003  12   ← 最高位址存最高位元組
```

### 範例：讀取部分資料

```asm
.data
myDouble DWORD 12345678h

.code
; mov ax, myDouble       ; 錯誤！大小不符（DWORD ≠ WORD）
mov ax, WORD PTR myDouble        ; AX = 5678h（低 word）
mov WORD PTR myDouble, 4321h     ; myDouble = 12344321h
```

### PTR 偏移存取

```asm
.data
myDouble DWORD 12345678h
; 等同於 myBytes BYTE 78h, 56h, 34h, 12h

.code
mov al, BYTE PTR myDouble        ; AL = 78h
mov al, BYTE PTR [myDouble+1]    ; AL = 56h
mov al, BYTE PTR [myDouble+2]    ; AL = 34h
mov ax, WORD PTR myDouble        ; AX = 5678h
mov ax, WORD PTR [myDouble+2]    ; AX = 1234h
```

### 將小資料搬入大目的運算元

```asm
.data
wordList WORD 5678h, 1234h
; 記憶體：78 56 34 12（小端序）

.code
mov eax, DWORD PTR wordList  ; EAX = 12345678h（CPU 自動反轉位元組）
```

### Your Turn 練習解析

```asm
.data
varB BYTE  65h,31h,02h,05h
varW WORD  6543h,1202h
varD DWORD 12345678h

.code
mov ax, WORD PTR [varB+2]  ; a. AX = 0502h
mov bl, BYTE PTR varD      ; b. BL = 78h
mov bl, BYTE PTR [varW+2]  ; c. BL = 02h
mov ax, WORD PTR [varD+2]  ; d. AX = 1234h
mov eax, DWORD PTR varW    ; e. EAX = 12026543h
```

## 4. TYPE 運算子

回傳單一元素的**大小（位元組數）**。

```asm
.data
var1 BYTE  ?   ; TYPE = 1
var2 WORD  ?   ; TYPE = 2
var3 DWORD ?   ; TYPE = 4
var4 QWORD ?   ; TYPE = 8

.code
mov eax, TYPE var1   ; 1
mov eax, TYPE var2   ; 2
mov eax, TYPE var3   ; 4
mov eax, TYPE var4   ; 8
```

---

## 5. LENGTHOF 運算子

計算**單一資料宣告中元素的數量**。

```asm
.data
byte1    BYTE  10,20,30           ; LENGTHOF = 3
array1   WORD  30 DUP(?),0,0      ; LENGTHOF = 30 + 2 = 32
array2   WORD  5 DUP(3 DUP(?))    ; LENGTHOF = 5 × 3 = 15
array3   DWORD 1,2,3,4            ; LENGTHOF = 4
digitStr BYTE  "12345678",0       ; LENGTHOF = 9

.code
mov ecx, LENGTHOF array1          ; ECX = 32
```

---

## 6. SIZEOF 運算子

回傳整個陣列佔用的**總位元組數** = `LENGTHOF × TYPE`。

```asm
.data
byte1    BYTE  10,20,30           ; SIZEOF = 3
array1   WORD  30 DUP(?),0,0      ; SIZEOF = 32 × 2 = 64
array2   WORD  5 DUP(3 DUP(?))    ; SIZEOF = 15 × 2 = 30
array3   DWORD 1,2,3,4            ; SIZEOF = 4 × 4 = 16
digitStr BYTE  "12345678",0       ; SIZEOF = 9

.code
mov ecx, SIZEOF array1            ; ECX = 64
```

## 7. 跨多行宣告（Spanning Multiple Lines）

### 同一宣告（逗號結尾）

```asm
.data
array WORD 10,20,
           30,40,    ; ← 逗號代表同一個宣告
           50,60

.code
mov eax, LENGTHOF array   ; 6
mov ebx, SIZEOF array     ; 12（6 × 2）
```

### 不同宣告（無逗號）

```asm
.data
array WORD 10,20
      WORD 30,40    ; ← 這是新的、獨立的宣告
      WORD 50,60

.code
mov eax, LENGTHOF array   ; 2（只計算第一行）
mov ebx, SIZEOF array     ; 4（2 × 2）
```

> **關鍵差異**：每行結尾有逗號 → 同一宣告；無逗號 → 新的獨立宣告，`LENGTHOF`/`SIZEOF` 只計算第一行。

## 8. LABEL 指令

為**既有的記憶體位置**指定另一個標籤名稱與型別，**不配置新的儲存空間**。

效果等同於 PTR，但宣告後使用更簡潔，不需每次寫 PTR。

```asm
.data
dwList   LABEL DWORD    ; dwList 與 wordList 指向同一記憶體
wordList LABEL WORD     ; wordList 與 intList 指向同一記憶體
intList  BYTE  00h,10h,00h,20h

; 記憶體內容（小端序）：00 10 00 20
;                        ↑
;                      intList / wordList / dwList 都指向這裡

.code
mov eax, dwList    ; EAX = 20001000h（讀取 4 bytes）
mov cx,  wordList  ; CX  = 1000h    （讀取 2 bytes）
mov dl,  intList   ; DL  = 00h      （讀取 1 byte）
```

## 三大運算子比較表

|運算子|回傳值|範例（`array DWORD 1,2,3,4`）|
|---|---|---|
|`TYPE`|單一元素大小（bytes）|`TYPE array` = **4**|
|`LENGTHOF`|元素個數|`LENGTHOF array` = **4**|
|`SIZEOF`|總位元組數|`SIZEOF array` = **16**|

> 恆等式：`SIZEOF = LENGTHOF × TYPE`
