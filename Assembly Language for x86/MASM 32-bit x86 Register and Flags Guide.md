在撰寫 MASM 程式時，選擇正確的暫存器與理解旗標的變化，是控制程式邏輯與記憶體存取的關鍵。

---

### 1. 通用暫存器 (General-Purpose Registers)

這八個 32 位元暫存器主要用於算術運算與資料搬移 。雖然稱為「通用」，但在某些特定指令（如乘除法、字串處理、迴圈）中，硬體會強制或預設使用特定的暫存器。

可以根據需要的資料大小，存取這些暫存器的不同部分 ：

|**32位元**|**16位元**|**8位元 (高)**|**8位元 (低)**|**全稱 (英文)**|**在 MASM 中的主要用途與特性**|
|---|---|---|---|---|---|
|**EAX**|AX|AH|AL|Accumulator|**累加暫存器**。常用於算術運算，且是乘法 (`MUL`/`IMUL`) 與除法 (`DIV`/`IDIV`) 指令的預設操作數 。函式的回傳值通常也會放在 EAX 中。|
|**EBX**|BX|BH|BL|Base|**基底暫存器**。通用性極高，常用作記憶體定址時的基底指標，或單純用來存放資料。|
|**ECX**|CX|CH|CL|Count|**計數暫存器**。是 `LOOP` 指令與字串重複指令 (`REP`) 的預設迴圈計數器 。在執行這些指令前，必須先將執行次數存入 ECX。|
|**EDX**|DX|DH|DL|Data|**資料暫存器**。在進行 32 位元乘除法時，常與 EAX 結合作為 64 位元的延伸暫存器 (EDX:EAX)。也常用於 I/O 埠的操作。|

此外，還有四個主要用於**指標與索引**的暫存器，它們**沒有 8 位元的分割名稱**，只能存取其 32 位元或 16 位元的低半部 ：

|**32位元**|**16位元**|**全稱 (英文)**|**在 MASM 中的主要用途與特性**|
|---|---|---|---|
|**ESI**|SI|Source Index|**來源索引暫存器**。主要用於高速的區塊資料或字串搬移操作（搭配字串指令如 `MOVSD`） 。預設指向 DS 區段。|
|**EDI**|DI|Destination Index|**目的索引暫存器**。與 ESI 配對使用，用於字串操作的目的地位址 。預設指向 ES 區段。|
|**ESP**|SP|Stack Pointer|**堆疊指標暫存器**。永遠指向當前堆疊的頂端 。**極不建議**將其當作一般運算暫存器使用，會導致堆疊崩潰。|
|**EBP**|BP|Base Pointer|**基底指標暫存器 (或稱延伸堆疊框指標)** 。用於在高階語言的函式呼叫中，建立「堆疊框 (Stack Frame)」，藉此安全且穩定地存取傳入的參數與區域變數。|

### 2. 區段暫存器 (Segment Registers)

在 16 位元真實模式 (Real-address mode) 下，這些暫存器儲存記憶體區段的基底位址。但在 32 位元保護模式 (Protected mode) 的「平坦記憶體模型 (Flat Memory Model)」下，MASM 開發者通常不需要手動更改它們，作業系統會自動處理 。

|**16位元**|**全稱 (英文)**|**用途與說明**|
|---|---|---|
|**CS**|Code Segment|**程式碼區段**。存放正在執行的指令機器碼 。|
|**DS**|Data Segment|**資料區段**。存放程式中的全域變數與靜態資料 。|
|**SS**|Stack Segment|**堆疊區段**。定義系統堆疊的記憶體範圍 。|
|**ES, FS, GS**|Extra Segments|**額外區段**。提供額外的資料區段指標 。在 Windows 保護模式下，FS 常被 OS 用來指向執行緒資訊區塊 (TEB)。|

---

### 3. 狀態旗標 (Status Flags)

旗標位於 **EFLAGS** 暫存器中。每一個旗標都是一個獨立的二進位位元 (bit) 。它們會反映 CPU 執行算術或邏輯運算（如 `ADD`, `SUB`, `CMP`, `AND`）後的結果 。 在 MASM 中，我們主要透過這些旗標來配合**條件跳躍指令**（例如 `JZ`, `JC`, `JG`）以實現 `if-else` 或 `while` 等邏輯控制。

|**旗標縮寫**|**全稱 (英文)**|**意義與觸發條件**|**MASM 中的應用情境**|
|---|---|---|---|
|**ZF**|Zero Flag|**零旗標**。如果運算結果等於零，則設為 1 。|最常用的旗標。常配合 `CMP` 指令判斷兩個值是否相等（相減結果為 0），並以 `JE` (Jump if Equal) 或 `JZ` (Jump if Zero) 進行跳躍。|
|**SF**|Sign Flag|**符號旗標**。如果運算結果為負數（最高位元為 1），則設為 1 。|處理**有號數** (Signed) 運算時，用來判斷結果的正負。|
|**CF**|Carry Flag|**進位旗標**。如果是**無號數** (Unsigned) 運算，且結果超出了目的地的範圍（發生進位或借位），則設為 1 。|處理無號數大小比較，或進行大整數的多重精度加法 (`ADC`)。對應的跳躍指令為 `JC` 或 `JB` (Jump if Below)。|
|**OF**|Overflow Flag|**溢位旗標**。如果是**有號數** (Signed) 運算，且結果超出了目的地的範圍，則設為 1 。|當兩個正數相加變成負數，或兩個負數相加變成正數時會觸發。常用於有號數的算術除錯。|
|**PF**|Parity Flag|**同位旗標**。如果運算結果的最低位元組 (LSB) 中，包含偶數個 1，則設為 1 。|早期用於資料傳輸錯誤檢查，現代程式撰寫中較少主動使用。|
|**AF**|Auxiliary Carry|**輔助進位旗標**。在二進位運算中，若第 3 位元向第 4 位元發生進位或借位，則設為 1 。|專門用於二進位編碼十進制 (BCD) 的算術運算調整。|

> **開發小提醒：** `CMP` 指令（比較）本質上就是執行減法運算，但**不會**把結果存回暫存器，它唯一的動作就是去改變上述的狀態旗標。接下來緊接的條件跳躍指令就是根據這些旗標來決定程式的走向。
