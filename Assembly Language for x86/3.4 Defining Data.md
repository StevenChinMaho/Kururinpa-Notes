### 1. 內建資料型別 (Intrinsic Data Types)

組合語言提供了多種基本的資料型別，用於定義不同大小的整數與實數 ：

- **整數型別**：
    
    - **BYTE / SBYTE**：8 位元不帶正負號 / 帶正負號整數 。
        
    - **WORD / SWORD**：16 位元不帶正負號 / 帶正負號整數 。
        
    - **DWORD / SDWORD**：32 位元不帶正負號 / 帶正負號整數 。
        
    - **QWORD**：64 位元整數 。
        
    - **TBYTE**：80 位元整數（常用於 BCD）。
        
- **實數 (浮點數) 型別**：
    
    - **REAL4**：4-byte IEEE 短實數 。
        
    - **REAL8**：8-byte IEEE 長實數 。
        
    - **REAL10**：10-byte IEEE 擴充實數 。

### 2. 資料定義陳述式 (Data Definition Statement)

資料定義陳述式會在記憶體中為變數保留儲存空間，並可選擇性地命名 。

- **語法格式**：`[名稱] 假指令 初始值 [, 初始值] ...` 。
    
- **初始值**：所有的初始值最終在記憶體中都會轉化為二進位資料 。可以使用 `?` 來表示不初始化該資料 。
    

---

### 3. 定義字串與陣列 (Strings and Arrays)

- **字串定義**：字串在組合語言中被視為**字元陣列**，通常以引號包覆並以空字元 (NULL, `0`) 結尾 。
    
    - 範例：`str1 BYTE "Enter your name",0` 。
        
    - **換行字元**：使用 `0Dh` (Carriage Return) 與 `0Ah` (Line Feed) 組合來達成換行 。
        
- **DUP 運算子**：用於重複分配空間（建立陣列或字串）。
    
    - 語法：`計數器 DUP (參數)` 。
        
    - 範例：`var1 BYTE 20 DUP (0)`（建立 20 個皆為 0 的 bytes）。
        

---

### 4. 特殊資料定義細節

- **BCD 資料 (TBYTE)**：Intel 以 10-byte 封裝儲存 BCD 整數 。
    
    - 每個 byte（最高位除外）包含兩個十進位數字 。
        
    - 最高 byte 的最高位元代表正負號（例如 `80h` 代表負數）。
        
- **小端序 (Little Endian Order)**：
    
    - 大於一個 byte 的資料型別，其個別 byte 會以**反向順序**儲存於記憶體中 。
        
    - **最低位元組 (LSB)** 會被放在第一個（最低的）記憶體位址 。
        
    - 例如：`12345678h` 在記憶體中的順序為 `78, 56, 34, 12` 。
        

---

### 5. 未初始化資料 (.data?)

- 使用 **`.data?`** 假指令來宣告一個不初始化的資料區段 。
    
- **優點**：可以**減少程式執行檔 (.EXE) 的大小** 。
    
    - 在 `.data` 中定義大陣列（如 `5000 DUP(0)`）會直接佔用執行檔空間 。
        
    - 在 `.data?` 中定義則不會在檔案中預佔空間，僅在載入記憶體時分配 。
        

---

### 6. 指令與輸入/輸出範例

- **讀取字串**：使用 `ReadString` 之前需要先設定 `edx` 為緩衝區位址，`ecx` 為緩衝區大小 。
    
- **顯示字串**：使用 `WriteString` 顯示 `edx` 所指向位址的字串 。
