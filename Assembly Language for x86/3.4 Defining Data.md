# 3.4 Defining Data

## 一、資料型態 (Intrinsic Data Types)

|型態|大小|說明|
|---|---|---|
|BYTE / SBYTE|8-bit|無號 / 有號整數|
|WORD / SWORD|16-bit|無號 / 有號整數|
|DWORD / SDWORD|32-bit|無號 / 有號整數|
|QWORD|64-bit|整數|
|TBYTE|80-bit|整數（BCD 用）|
|REAL4|4 bytes|IEEE 短實數|
|REAL8|8 bytes|IEEE 長實數|
|REAL10|10 bytes|IEEE 延伸實數|

---

## 二、資料定義語法

```asm
[name] directive initializer [,initializer] ...
```

範例：

```asm
value1 BYTE 10       ; 單一初始值
value2 BYTE ?        ; 未初始化
myList BYTE 10,20,30 ; 多個初始值
```

所有初始值最終都以**二進位**形式存在記憶體中。

---

## 三、定義各型態資料

### BYTE / SBYTE

```asm
value1 BYTE 'A'        ; 字元常數 (= 65)
value2 BYTE 0          ; 最小無號值
value3 BYTE 255        ; 最大無號值
value4 SBYTE -128      ; 最小有號值
value5 SBYTE +127      ; 最大有號值
value6 BYTE ?          ; 未初始化
```

### WORD / SWORD

```asm
word1 WORD  65535
word2 SWORD -32768
word3 WORD  "AB"       ; 雙字元
```

### DWORD / SDWORD

```asm
val1 DWORD  12345678h
val2 SDWORD -2147483648
val3 DWORD  20 DUP(?)  ; 陣列
```

### QWORD / TBYTE / REAL

```asm
quad1 QWORD  1234567812345678h
rVal1 REAL4  -1.2
rVal2 REAL8  3.2E-260
rVal3 REAL10 4.6E+4096
```

---

## 四、字串定義

字串是**字元陣列**，通常以 null (0) 結尾：

```asm
str1 BYTE "Enter your name", 0
str2 BYTE 'Error: halting', 0
```

跨行字串（每行末尾加逗號）：

```asm
menu BYTE "Option 1", 0dh, 0ah,
     "Option 2", 0dh, 0ah, 0
```

換行序列：

- `0Dh` = Carriage Return（歸位）
- `0Ah` = Line Feed（換行）
- 合起來相當於 C 語言的 `'\n'`

---

## 五、DUP 運算子

用於建立**陣列或重複字串**：

```asm
; 語法：counter DUP(argument)

var1 BYTE 20 DUP(0)        ; 20 bytes，全為 0
var2 BYTE 2  DUP(?)        ; 2 bytes，未初始化
var3 BYTE 4  DUP("STACK")  ; 20 bytes："STACKSTACKSTACKSTACK"
var4 BYTE 10, 3 DUP(0), 20 ; 5 bytes
```

---

## 六、Little Endian 儲存順序

x86 將**最低有效位元組存在最低位址**：

```asm
val1 DWORD 12345678h
```

|位址|值|
|---|---|
|0000|78|
|0001|56|
|0002|34|
|0003|12|

> 與 Big Endian（網路位元組順序）相反。

---

## 七、BCD 資料 (TBYTE)

Intel 以 10 bytes 儲存 Packed BCD，最高位元組的最高位元表示正負號：

```asm
; +1234 → 34 12 00 00 00 00 00 00 00 00
; -1234 → 34 12 00 00 00 00 00 00 00 80

intVal TBYTE 80000000000000001234h  ; 表示 -1234
```

初始值**必須以十六進位**表示，不能直接寫負數。

---

## 八、未初始化資料 (.data?)

使用 `.data?` 區段宣告未初始化變數，可**減少 EXE 檔案大小**：

```asm
.data
  smallArray DWORD 10 DUP(0)    ; 40 bytes（納入 EXE）

.data?
  bigArray DWORD 5000 DUP(?)    ; 20000 bytes（不納入 EXE）
```

若把 `bigArray` 放在 `.data` 中，EXE 會多出 20,000 bytes。

---

## 九、重點整理

```
資料型態大小：BYTE(1) < WORD(2) < DWORD(4) < QWORD(8) < TBYTE(10)
? 表示未初始化
DUP 可建立陣列
字串需自行加 null 結尾 (0)
x86 使用 Little Endian
.data? 可節省 EXE 大小
```
