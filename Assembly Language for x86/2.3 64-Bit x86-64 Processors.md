### 1. 指令集與基本特性 (Instruction Set)

- **架構定義**：x86-64 是 x86 指令集的 64 位元延伸版本 。
    
- **向下相容**：此架構與現有的 x86 指令集保持向下相容 。
    
- **定址能力**：
    
    - 雖然位址長度為 64 位元，但目前的處理器僅實際使用其中的 **48 位元** 。
        
    - 可支援高達 **256 TB** 的定址空間 。
        
- **暫存器升級**：通用暫存器全部提升為 64 位元 。
    

### 2. 運作模式 (Operation Modes)

x86-64 引入了稱為 **IA32e** 的新模式，其下包含兩個子模式 ：

- **64 位元模式 (64-bit Mode)**：
    
    - 啟用 x86-64 指令集 。
        
    - 可使用 64 位元線性位址空間 。
        
    - 在原生 64 位元模式下運作時，**不支援** 16 位元真實模式或虛擬-8086 模式 。
        
- **相容模式 (Compatibility Mode)**：
    
    - 通常可以執行 16 位元與 32 位元應用程式 。
        
    - **限制**：在 64 位元 Windows 下，並不支援傳統的 16 位元程式碼 。
        
    - 64 位元 Windows 沒有虛擬 DOS 機器 (VDM) 子系統 。
        

### 3. 基本執行環境：暫存器 (Registers)

相較於 32 位元架構，64 位元環境在暫存器數量與大小上都有顯著增加：

- **通用暫存器 (General Purpose Registers)**：
    
    - 數量從 8 個增加到 **16 個** 64 位元暫存器 。
        
- **狀態旗標暫存器 (RFLAGS)**：
    
    - 寬度為 64 位元，但目前僅使用**低 32 位元** 。
        
- **指令指標暫存器 (RIP)**：
    
    - 升級為 64 位元暫存器 。
        
- **浮點與 SIMD 暫存器**：
    
    - 維持 8 個 80 位元浮點暫存器 。
        
    - 維持 8 個 64 位元 MMX 暫存器 。
        
    - **XMM 暫存器**：從 8 個增加到 **16 個** 128 位元暫存器 。
