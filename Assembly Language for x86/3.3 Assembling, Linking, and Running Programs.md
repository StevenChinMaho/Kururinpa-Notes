### 1. 組譯 - 連結 - 執行 週期 (Assemble-Link Execute Cycle)

這個週期描述了從建立原始程式碼到執行編譯後程式的完整步驟 。請注意，如果修改了原始碼 (Source code)，就必須重複步驟 2 到步驟 4 。

完整的流程分為以下四個步驟：

- **步驟 1：文字編輯器 (Text editor)**
    
    - 開發者使用文字編輯器撰寫程式碼，產生 **原始檔 (Source File)** 。
        
- **步驟 2：組譯器 (Assembler)**
    
    - 組譯器讀取原始檔，並將其轉換成 **目的檔 (Object File)** 。
        
    - 同時，組譯器也可以選擇性地產生 **列表檔 (Listing File)**，用於檢視編譯細節 。
        
- **步驟 3：連結器 (Linker)**
    
    - 連結器將目的檔與 **連結函式庫 (Link Library)** 結合 。
        
    - 連結完成後會產生最終的 **執行檔 (Executable File)**，並可選擇性地產生 **對應檔 (Map File)** 。
        
- **步驟 4：作業系統載入器 (OS loader)**
    
    - 作業系統的載入器會讀取執行檔，將其載入記憶體中執行，最後產生 **輸出結果 (Output)** 。

### 2. 列表檔 (Listing File)

列表檔是一個非常有用的工具，可以讓開發者**查看程式是如何被編譯的** 。

列表檔內部包含以下重要資訊：

- **原始碼 (source code)**：你所撰寫的組合語言程式碼 。
    
- **位址 (addresses)**：各指令與資料在記憶體中的相對位址 。
    
- **目的碼 (object code)**：即翻譯過後的機器語言 (machine language) 。
    
- **區段名稱 (segment names)**：如 code、data 等區段的名稱 。
    
- **符號 (symbols)**：包含程式中定義的變數 (variables)、程序 (procedures) 與常數 (constants) 。

### 3. 在 Visual Studio 中設定產生列表檔 (Set MASM to generate list file)

如果你使用 Visual Studio 開發，可以透過修改專案屬性來強制 MASM 產生列表檔。設定步驟如下：

1. 開啟專案的 **屬性頁 (Property Pages)** 。
    
2. 確認上方的「組態 (Configuration)」為 **作用中 (Debug)**，且「平台 (Platform)」為 **Win32** 。
    
3. 在左側選單中，依序展開並選擇：
    
    - **Microsoft Macro Assembler**
        
    - **Listing File**
        
4. 在右側的設定項目中，找到 **Assembled Code Listing File**（產生組譯程式碼列表檔） 。
    
5. 將其值設定為 `$(ProjectName).lst` 。
    
6. 這個設定在底層相當於加入了命令列參數 `/Fl[file]` 。完成後點擊「套用」與「確定」即可 。
