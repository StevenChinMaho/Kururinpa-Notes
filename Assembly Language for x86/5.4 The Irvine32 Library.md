# The Irvine32 Library（Irvine32 程式庫）

> Assembly Language for x86 Processors (7th Edition) — Chapter 5: Procedures

---

## 為什麼需要 Irvine32 Library？

在組合語言中，即使是「在螢幕上顯示一個整數」這樣簡單的任務，底層實作也相當複雜（需要反覆取餘數、轉換 ASCII、處理正負號等）。Irvine32 Library 將這些繁瑣的 I/O 與資料轉換操作封裝成易用的程序介面，讓初學者能專注在語言本身的學習。

---

## 使用方式

在程式開頭加入：

```asm
INCLUDE Irvine32.inc    ; 引入所有程序的 PROTO 宣告
```

並在專案連結設定中加入 `Irvine32.lib`。

## 完整程序總覽

### I — 螢幕清除與游標控制

| 程序 | 輸入 | 輸出 / 行為 |
|---|---|---|
| `Clrscr` | 無 | 清除 console 畫面，游標移到左上角 |
| `Gotoxy` | DH = 列（row），DL = 欄（col） | 將游標移到指定位置 |
| `GetMaxXY` | 無 | 取得 console 視窗的欄數與列數 |
| `GetTextColor` | 無 | 回傳目前前景與背景顏色 |
| `SetTextColor` | EAX = 顏色代碼 | 設定後續輸出的前景與背景顏色 |

**SetTextColor 顏色計算**：

```asm
; 顏色代碼 = 前景色 + (背景色 × 16)
mov eax, yellow + (blue * 16)   ; 藍底黃字
call SetTextColor
```

AL 的位元配置：

```
[ 高 4 bits：背景色 | 低 4 bits：前景色 ]
```

---

### II — 字元與字串輸出

| 程序 | 輸入 | 輸出 / 行為 |
|---|---|---|
| `WriteChar` | AL = 字元 ASCII 碼 | 輸出單一字元到 console |
| `WriteString` | EDX = 字串起始位址（null-terminated） | 輸出字串到 console |
| `Crlf` | 無 | 輸出換行（CR/LF），游標移到下一行開頭 |

**WriteString 範例**：

```asm
.data
str1 BYTE "Assembly language is easy!", 0

.code
mov edx, OFFSET str1
call WriteString    ; 輸出字串
call Crlf           ; 換行
```

**嵌入式換行（Embedded CR/LF）**：另一種換行方式，直接在字串資料中嵌入 `0Dh, 0Ah`：

```asm
str1 BYTE "Assembly language is easy!", 0Dh, 0Ah, 0
; 只需 call WriteString，不需額外 call Crlf
```

---

### III — 整數輸出

| 程序 | 輸入 | 輸出格式 |
|---|---|---|
| `WriteInt` | EAX = 有號 32-bit 整數 | 十進位（帶正負號，如 `+216543`） |
| `WriteDec` | EAX = 無號 32-bit 整數 | 十進位（無正負號） |
| `WriteHex` | EAX = 無號 32-bit 整數 | 十六進位（8 位，如 `0000001F`） |
| `WriteHexB` | EAX = 整數 | 十六進位，可指定 1/2/4/8 byte 格式 |
| `WriteBin` | EAX = 無號 32-bit 整數 | 二進位（32 位，如 `0000 0000 ... 0010 0011`） |
| `WriteBinB` | EAX = 整數 | 二進位，可指定 byte/word/dword 格式 |

**多格式顯示同一整數範例**：

```asm
IntVal = 35
.code
    mov eax, IntVal
    call WriteBin       ; 0000 0000 0000 0000 0000 0000 0010 0011
    call Crlf
    call WriteDec       ; 35
    call Crlf
    call WriteHex       ; 00000023
    call Crlf
```

### IV — 字串與整數輸入

| 程序 | 輸入參數 | 回傳 / 行為 |
|---|---|---|
| `ReadChar` | 無 | AL = 讀取的單一字元 |
| `ReadString` | EDX = 緩衝區位址，ECX = 最大字元數 | 讀取一行字串（Enter 結束），自動附加 null byte |
| `ReadInt` | 無 | EAX = 讀入的 32-bit 有號整數 |
| `ReadDec` | 無 | EAX = 讀入的 32-bit 無號整數 |
| `ReadHex` | 無 | EAX = 讀入的 32-bit 十六進位整數 |
| `ReadKey` | 無 | 從鍵盤輸入緩衝區讀取字元（不等待） |

**ReadString 範例**：

```asm
.data
fileName BYTE 80 DUP(0)

.code
mov edx, OFFSET fileName
mov ecx, SIZEOF fileName - 1   ; 最多讀 79 個字元（保留 null byte 空間）
call ReadString
```

### VII — 隨機數

| 程序 | 說明 |
|---|---|
| `Randomize` | 以當前時間為種子，初始化亂數產生器（建議程式開頭呼叫一次） |
| `Random32` | 回傳 0 ~ FFFFFFFFh 範圍的 32-bit 虛擬亂數（存於 EAX） |
| `RandomRange` | EAX = 上限值（ceiling），回傳 0 ~ EAX-1 的亂數（存於 EAX） |

**RandomRange 範例**：

```asm
mov ecx, 10     ; 迴圈 10 次
L1:
    mov eax, 100        ; 上限 100
    call RandomRange    ; EAX = 0~99 的亂數
    call WriteInt       ; 顯示（有號格式）
    call Crlf
    loop L1
```

---

### VIII — 時間延遲

| 程序 | 輸入 | 說明 |
|---|---|---|
| `Delay` | EAX = 毫秒數 | 暫停程式執行指定毫秒 |
| `GetMseconds` | 無 | 回傳自午夜起經過的毫秒數（EAX） |

```asm
mov eax, 500    ; 暫停 500 毫秒
call Delay
```

### IX — 記憶體與暫存器除錯輸出

| 程序 | 輸入 | 說明 |
|---|---|---|
| `DumpMem` | ESI = 起始位址，ECX = 元素個數，EBX = 元素大小（bytes） | 以十六進位顯示一段記憶體內容 |
| `DumpRegs` | 無 | 顯示所有通用暫存器與旗標的十六進位值 |

**DumpMem 範例**：

```asm
mov esi, OFFSET arrayD
mov ebx, TYPE arrayD      ; 元素大小（DWORD = 4）
mov ecx, LENGTHOF arrayD  ; 元素個數
call DumpMem
```

輸出範例：

```
Dump of offset 00405000
-------------------------------
12345678  1A4B2000  00003434  00007AB9
```

**DumpRegs 輸出範例**：

```
EAX=00000613  EBX=00000000  ECX=000000FF  EDX=00000000
ESI=00000000  EDI=00000100  EBP=0000091E  ESP=000000F6
EIP=00401026  EFL=00000286  CF=0  SF=1  ZF=0  OF=0
```

| 讀入有號整數 | `ReadInt` | 回傳於 EAX |

## 重點整理

- Irvine32 Library 封裝了 x86 環境下繁瑣的 I/O 操作，讓初學者可用簡單的 `CALL` 指令完成輸入輸出。
- 使用前需在程式開頭加 `INCLUDE Irvine32.inc`，並在連結設定中加入 `Irvine32.lib`。
- **引數幾乎都透過暫存器傳遞**：字串位址放 EDX、整數值放 EAX、計數放 ECX。
- **回傳值通常在 EAX**：ReadInt、RandomRange、StrLength 等程序均如此。
- 顏色設定公式：`EAX = 前景色 + (背景色 × 16)`，在 `SetTextColor` 之後呼叫 `Clrscr` 才能讓背景色生效。
