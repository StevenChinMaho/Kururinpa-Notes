## 本節涵蓋主題

|主題|說明|
|---|---|
|間接運算元（Indirect Operands）|用暫存器當作指標存取記憶體|
|陣列（Arrays）|用間接定址走訪陣列元素|
|索引運算元（Indexed Operands）|常數 + 暫存器產生有效位址|
|比例因子（Scale Factor）|索引值乘以元素大小來定址|
|指標（Pointers）|用變數儲存另一個變數的位址|

---

## 1. 間接運算元（Indirect Operands）

用 32 位元通用暫存器（EAX、EBX、ECX、EDX、ESI、EDI、EBP、ESP）作為指標，以**中括號 `[ ]`** 取值（dereference）。

```asm
.data
byteVal BYTE 10h

.code
mov esi, OFFSET byteVal   ; ESI = byteVal 的位址
mov al, [esi]             ; 讀取 ESI 所指記憶體的值 → AL = 10h
mov bl, 3
mov [esi], bl             ; 將 3 寫入 ESI 所指的記憶體位置
```

### 解參考示意

```
ESI = 10400h
記憶體[10400h] = 10h
mov al, [esi]  →  AL = 10h
```

### 搭配 PTR 消除大小歧義

組譯器**不追蹤**暫存器指向的型別，若對 `[esi]` 直接進行 `inc` 等操作會產生歧義錯誤，必須用 PTR 明確指定大小。

```asm
.data
myCount WORD 0,1,2

.code
mov esi, OFFSET myCount
inc [esi]             ; 錯誤：ambiguous（不知道是 BYTE、WORD 還是 DWORD）
inc WORD PTR [esi]    ; 正確：對 ESI 所指位置的 WORD 值加 1

add WORD PTR [esi], 20  ; 同樣需要 PTR
```

## 2. 陣列（Arrays）

間接定址最適合用來**逐步走訪陣列**。每次移動的步距必須與陣列元素型別的大小一致。

|陣列型別|每次步距|
|---|---|
|`BYTE`|+1|
|`WORD`|+2|
|`DWORD`|+4|

### WORD 陣列範例

```asm
.data
arrayW WORD 1000h, 2000h, 3000h
;  記憶體佈局：
;  10200h → 1000h
;  10202h → 2000h
;  10204h → 3000h

.code
mov esi, OFFSET arrayW
mov ax, [esi]      ; AX = 1000h
add esi, 2         ; 或 add esi, TYPE arrayW（更具可讀性）
mov ax, [esi]      ; AX = 2000h
add esi, 2
mov ax, [esi]      ; AX = 3000h
```

### DWORD 陣列：累加三個整數

```asm
.data
arrayD DWORD 1000h, 2000h, 3000h

.code
mov esi, OFFSET arrayD
mov eax, [esi]     ; EAX = 00001000h（第一個元素）
add esi, 4
add eax, [esi]     ; EAX = 00003000h（加第二個）
add esi, 4
add eax, [esi]     ; EAX = 00006000h（加第三個）
```

> **注意**：若讀取超出陣列範圍的位址（如 `[esi+6]` 存取 WORD 陣列末端之後），間接定址會自動以 0 填入未宣告區域。

## 3. 索引運算元（Indexed Operands）

將**常數（變數名稱）與暫存器相加**來產生有效位址（Effective Address）。有兩種等效的寫法：

```asm
arrayB[esi]       ←→     [arrayB + esi]
arrayD[ebx]       ←→     [arrayD + ebx]
```

### 第一種：索引從 0 開始

初始化索引暫存器為 0，每次存取後手動遞增。

```asm
.data
arrayB BYTE 10h, 20h, 30h

.code
mov esi, 0
mov al, arrayB[esi]     ; AL = 10h（也可寫成 mov al, [arrayB+esi]）
```

### 第二種：暫存器存起始位址，常數為位移量

```asm
.data
arrayW WORD 1000h, 2000h, 3000h

.code
mov esi, OFFSET arrayW
mov ax, [esi]      ; AX = 1000h（等同 [arrayW]）
mov ax, [esi+2]    ; AX = 2000h（等同 [arrayW+2]）
mov ax, [esi+4]    ; AX = 3000h（等同 [arrayW+4]）
```

### 16 位元模式（Real-Address Mode）

在真實定址模式下，僅能使用 **SI、DI、BX、BP** 四個暫存器進行索引定址。

```asm
mov al, arrayB[si]
mov ax, arrayW[di]
mov ax, arrayD[bx]
```

> **注意**：避免使用 BP（Base Pointer），BP 主要用於存取堆疊中的資料。

## 4. 比例因子（Scale Factor）

將索引值乘以元素大小（TYPE），直接用邏輯索引（subscript）存取陣列元素，不必手動換算位元組偏移量。

### 寫法一：索引 × 常數

```asm
.data
arrayD DWORD 1, 2, 3, 4

.code
mov esi, 3                   ; 邏輯索引（第 4 個元素，從 0 起算）
mov eax, arrayD[esi*4]       ; EAX = 4（= arrayD[3]）
```

### 寫法二：使用 TYPE（推薦，可讀性高）

```asm
mov esi, 3
mov eax, arrayD[esi * TYPE arrayD]   ; EAX = 4，更具可移植性
```

### 寫法三：先計算偏移量存入索引暫存器

```asm
mov esi, 3 * TYPE arrayD     ; ESI = 12（= 3 × 4）
mov eax, arrayD[esi]         ; EAX = 4
```

## 5. 指標（Pointers）

指標是一個**儲存另一個變數位址**的變數。在 32 位元保護模式下，指標大小為 4 bytes（DWORD）。

### 基本宣告

```asm
.data
arrayB BYTE 10h, 20h, 30h, 40h
ptrB   DWORD arrayB           ; ptrB 儲存 arrayB 的起始位址

; 等效寫法：
ptrB DWORD OFFSET arrayB
```

### 使用指標存取資料

```asm
.code
mov esi, ptrB        ; ESI = arrayB 的位址（如 00001234h）
mov eax, [esi]       ; EAX = 40302010h（讀取 4 bytes，小端序）
```

## 6. TYPEDEF 運算子（自訂指標型別）

`TYPEDEF` 可以建立使用者自訂型別，讓指標宣告更具語意。

> 語法：`新型別名稱 TYPEDEF PTR 元素型別`（注意：`PTR BYTE`，不是 `BYTE PTR`）

### 宣告自訂指標型別

```asm
PBYTE  TYPEDEF PTR BYTE    ; 指向 BYTE 的指標
PWORD  TYPEDEF PTR WORD    ; 指向 WORD 的指標
PDWORD TYPEDEF PTR DWORD   ; 指向 DWORD 的指標
```

### 完整範例

```asm
.data
arrayB BYTE  10h, 20h, 30h
arrayW WORD  1, 2, 3
arrayD DWORD 4, 5, 6

; 使用自訂型別宣告指標變數
ptr1 PBYTE  arrayB    ; 指向 arrayB
ptr2 PWORD  arrayW    ; 指向 arrayW
ptr3 PDWORD arrayD    ; 指向 arrayD

.code
main PROC
    mov esi, ptr1
    mov al,  [esi]       ; AL  = 10h

    mov esi, ptr2
    mov ax,  [esi+2]     ; AX  = 2

    mov esi, ptr3
    mov eax, [esi]       ; EAX = 4
main ENDP
```

## 綜合範例

### 範例一：用 LOOP 累加 DWORD 陣列

搭配 `OFFSET`、`LENGTHOF`、`TYPE` 與 `LOOP` 指令，是走訪陣列的標準寫法。

```asm
.data
intarray DWORD 10000h, 20000h, 30000h, 40000h

.code
mov edi, OFFSET intarray     ; EDI 指向陣列起始
mov ecx, LENGTHOF intarray   ; ECX = 4（迴圈次數）
mov eax, 0                   ; 清空累加器
L1:
    add eax, [edi]           ; 累加目前元素
    add edi, TYPE intarray   ; EDI += 4，指向下一個元素
    loop L1                  ; ECX -= 1；若 ECX ≠ 0 則跳回 L1
; 結束：EAX = A0000h
```

> 備註：LOOP 與 ECX 會自動配合，迴圈每執行一次，ECX 會自動減 1，無須手動遞減。

### 範例二：複製字串

```asm
.data
source BYTE "This is the source string", 0
target BYTE SIZEOF source DUP(0)   ; 配置與 source 相同大小的空間

.code
mov esi, 0                   ; 索引從 0 開始
mov ecx, SIZEOF source       ; 迴圈次數 = 字串總長（含 null）
L1:
    mov al, source[esi]      ; 從 source 取一個字元
    mov target[esi], al      ; 寫入 target
    inc esi                  ; 移到下一個字元
    loop L1
```

## 重點整理

|寫法|說明|
|---|---|
|`[esi]`|以 ESI 的值作為記憶體位址取值（一定要加中括號）|
|`WORD PTR [esi]`|明確指定存取大小，避免歧義錯誤|
|`arrayB[esi]`|索引定址第一種形式（等同 `[arrayB+esi]`）|
|`arrayD[esi*4]`|比例因子：索引 × 型別大小|
|`add esi, TYPE arrayW`|移動步距應與元素型別大小一致|
|`TYPEDEF PTR BYTE`|建立語意明確的自訂指標型別|
