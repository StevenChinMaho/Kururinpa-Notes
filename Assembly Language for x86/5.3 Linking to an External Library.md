## 1. Library 是什麼？

**Library（程式庫）** 是一個包含一或多個已編譯程序（機器碼）的檔案，副檔名通常為 `.lib`。

- 由一或多個 **OBJ 檔**組合而成。
- 使用 Microsoft **LIB 工具**打包。
- 使用者只需知道程序的介面（名稱、輸入、輸出），不需看到原始碼。

**建立 Library 的流程**：

```
原始碼（.asm / .inc）
        ↓  組譯（ML.exe）
    目的檔（.obj）
        ↓  打包（LIB.exe）
    程式庫（.lib）
```

---

## 2. 呼叫 Library 程序

呼叫外部 Library 程序需要兩個步驟：

### 步驟一：在原始碼中宣告 PROTO

`PROTO` Directive 告訴組譯器「這個程序存在於外部 Library 中」，讓組譯器知道如何處理 `CALL`。

```asm
WriteString PROTO    ; 宣告 WriteString 存在於外部 Library
```

> 實際使用 Irvine32 時，只需在程式開頭加上 `INCLUDE Irvine32.inc`，  
> 該 `.inc` 檔已包含所有程序的 PROTO 宣告，不需手動逐一宣告。

### 步驟二：用 CALL 指令呼叫

```asm
call WriteString     ; 呼叫 Library 中的程序
```

---

## 3. Linker 的角色

**Linker** 在組譯完成後執行，負責將程式的 OBJ 檔與所需的 LIB 檔合併成最終的可執行檔（`.exe`）。

```
Linker 命令列範例：
link hello.obj irvine32.lib kernel32.lib
```

上述命令將 `hello.obj` 連結到 `irvine32.lib` 和 `kernel32.lib`，產生 `hello.exe`。

---

## 4. 32-bit 程式的連結架構

```
你的程式（.exe）
    ↓  links to
Irvine32.lib
    ↓  links to
kernel32.lib        ← Microsoft Win32 SDK 的一部分
    ↓  executes via
kernel32.dll        ← MS-Windows 的核心元件
```

| 檔案 | 說明 |
|---|---|
| `Irvine32.lib` | 課程使用的輔助 Library，封裝 I/O 等常用程序 |
| `kernel32.lib` | Win32 SDK 的連結資訊檔，記錄 kernel32.dll 中函式的位置 |
| `kernel32.dll` | Windows 作業系統的核心 DLL，提供底層 API |

> `kernel32.dll` 是 MS-Windows 的基礎組件，幾乎所有 Windows 程式都會連結到它。

## 5. 自建 Library 流程（以 Irvine32 環境為例）

若想新增自訂程序到 Library，可依照以下步驟：

### 步驟一：在 `Irvine32.asm` 中加入新程序

```asm
; 在 Irvine32.asm 中新增 ADDSUM 程序
ADDSUM PROC
; Receives: EAX, EBX, ECX
; Returns:  EAX = 三數總和
;-----------------------------------------
    .code
    add eax, ebx
    add eax, ecx
    ret
ADDSUM ENDP
```

### 步驟二：在 `Irvine32.inc` 中新增 PROTO 宣告

```asm
; 在 Irvine32.inc 的 Procedure Prototypes 區段加入：
ADDSUM PROTO    ; 讓使用者程式可以呼叫
```

### 步驟三：執行 `makeLib.bat` 重新建置 Library

```bat
REM makeLib.bat 的核心步驟（簡化）：
ML -c -coff Irvine32.asm          REM 組譯成 OBJ
LIB /SUBSYSTEM:CONSOLE Irvine32.obj floatio.obj  REM 打包成 LIB
```

> 所需工具（`ml.exe`、`lib.exe`、`link.exe`）位於 Visual Studio 的 VC\bin 資料夾。  
> 執行完畢後會產生新的 `Irvine32.lib`。

### 步驟四：在 Visual Studio 專案中加入 Library 參照

若出現 `error LNK2019: 無法解析的外部符號` 錯誤，需在專案屬性中手動加入 Library：

```
專案屬性 → 連結器 → 輸入 → 其他相依性
加入：mylib.lib;Irvine32.lib;%(AdditionalDependencies)
```

## 6. 建立自訂 Library（mylib）的完整結構

| 檔案 | 用途 |
|---|---|
| `mylib.asm` | 包含自訂程序的原始碼，開頭 `INCLUDE mylib.inc` |
| `mylib.inc` | 包含所有自訂程序的 PROTO 宣告 |
| `mymakeLib.bat` | 組譯 + 打包的批次腳本 |

**mylib.asm 結構**：

```asm
INCLUDE mylib.inc    ; 引入 PROTO 宣告

.code

;-------------------------------------------------
ADDSUM PROC
; Receives: EAX, EBX, ECX
; Returns:  EAX = 總和
;-------------------------------------------------
    add eax, ebx
    add eax, ecx
    ret
ADDSUM ENDP

END
```

**mylib.inc 結構**：

```asm
INCLUDE SmallWin.inc    ; MS-Windows 相關定義

; Procedure Prototypes
ADDSUM PROTO            ; 宣告自訂程序
```

**mymakeLib.bat 核心內容**：

```bat
ML -c -coff mylib.asm           REM 組譯 mylib.asm → mylib.obj
LIB /SUBSYSTEM:CONSOLE mylib.obj floatio.obj  REM 打包成 mylib.lib
```

**使用者程式引用自訂 Library**：

```asm
;INCLUDE Irvine32.inc    ; 若不需要 Irvine32 可注解掉
INCLUDE mylib.inc        ; 引入自訂 Library 的 PROTO 宣告

.code
main PROC
    mov eax, 10
    mov ebx, 12
    call ADDSUM          ; 呼叫自訂程序，結果在 EAX
    Invoke ExitProcess, 0
main ENDP
END main
```

## 重點整理

| 概念 | 說明 | 關鍵注意事項 |
|---|---|---|
| Library（`.lib`） | 已編譯程序的集合檔 | 由 OBJ 檔打包而成 |
| `PROTO` Directive | 宣告外部程序的存在 | 使用 `.inc` 可批次引入所有 PROTO |
| `INCLUDE` Directive | 將 `.inc` 檔內容複製進來 | 通常放在程式最開頭 |
| `CALL` | 呼叫 Library 中的程序 | 與呼叫自訂程序語法完全相同 |
| Linker | 合併 OBJ 與 LIB，產生 EXE | 需明確指定要連結哪些 LIB |
| `kernel32.lib` | Win32 SDK 連結資訊 | 對應 `kernel32.dll`，是 Windows 底層 API |
| LNK2019 錯誤 | 找不到外部符號 | 在專案屬性「其他相依性」加入 `.lib` 名稱 |
| 自建 Library 流程 | `.asm` → `.obj` → `.lib` | 需同步更新 `.inc` 中的 PROTO 宣告 |
