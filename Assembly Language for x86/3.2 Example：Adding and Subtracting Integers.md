# 3.2 Example：Adding and Subtracting Integers

## 1. 範例程式碼：兩個整數相加 (Example: Adding Two Integers)

這是一個基本的 32 位元整數相加程式（AddTwo.asm）的架構 ：

- `.386`：表示允許存取 32 位元的暫存器與位址 。
    
- `.model flat, stdcall`：設定記憶體模式與呼叫慣例 。
    
- `.stack 4096`：設定堆疊大小 。
    
- `ExitProcess PROTO, dwExitCode: DWORD`：宣告 `ExitProcess` 這個 Windows 服務函數，並標示其輸入參數（input parameter）為 DWORD 型態 。
    
- 主程式區段（`.code`）包含將 5 移入 `eax`，再將 6 加到 `eax`，最後呼叫 `ExitProcess,0` 來結束程式 。
    
- `END main`：標示程式碼的結尾 。
    

## 2. 組譯器假指令回顧 (A Review of the Assembler Directives)

- **`.model flat, stdcall`** ：
    
    - **`flat`**：我們在 32 位元程式（保護模式，protected mode）中總是使用 flat 記憶體模式。這決定了程式碼和資料容量的大小限制 。
        
    - **`stdcall`**：告訴編譯器在呼叫程序 (procedure) 時，執行時期堆疊 (run-time stack) 應該如何被使用 。
        
- **`.stack 4096`**：為執行時期堆疊 (runtime stack) 保留 4096 bytes 的儲存空間 。
    
- **`End main`**：代表程式的結束，並參考（指向）程式的進入點 (program entry point) 。
    

## 3. 記憶體模式總覽 (Memory Models)

以下是各種記憶體模式的詳細規範與限制 ：

|**模式 (Model)**|**描述 (Description)**|
|---|---|
|**Tiny**|程式碼與資料的總和必須小於 64K。|
|**Small**|程式碼 <= 64K，資料 <= 64K。包含一個程式碼區段 (code segment) 與一個資料區段 (data segment)。|
|**Medium**|資料 <= 64K，程式碼大小不限。包含多個程式碼區段，一個資料區段。|
|**Compact**|程式碼 <= 64K，資料大小不限。包含一個程式碼區段，多個資料區段。|
|**Large**|程式碼 > 64K，資料 > 64K。包含多個程式碼與資料區段。|
|**Huge**|與 Large 模式相同，但個別變數（例如陣列）的容量可以大於 64K。|
|**Flat**|沒有區段 (No segments)。程式碼與資料均使用 32 位元位址。僅適用於保護模式 (Protected mode only)。|

## 4. 執行與偵錯 (Running and Debugging the AddTwo Program)

在偵錯模式下，建議學習以下操作來觀察程式運行 ：

- 學習如何在除錯器中使用**逐步執行 (Step Over)** 。
    
- 觀察**暫存器數值 (register values)** 的變化 。
    
- 觀察**狀態旗標 (status flags)** 是如何改變的 。
    

## 5. 建議的程式碼撰寫標準 (Suggested Coding Standards)

為了讓程式碼更易讀，投影片建議了以下風格規範：

- **大小寫的選擇 (Capitalization)** ：
    
    - 可以選擇全小寫、全大寫，或將所有保留字（包含指令助憶碼、暫存器名稱）大寫，也可以只將假指令與運算子大寫 。重點是保持一致。
        
- **命名與排版 (Naming & Spacing)** ：
    
    - 使用具描述性的識別字名稱 (descriptive identifier names) 。
        
    - 在算術運算子周圍加上空格 。
        
    - 在不同程序 (procedures) 之間保留 1 到 2 行的空白行 。
        
    - 程式碼與資料標籤 (labels) **不縮排** 。
        
    - 可執行的指令縮排 4 到 5 個空白 。
        
    - 指令與其運算元之間保留 1 到 3 個空白（例如：`mov ax,bx`） 。
        
    - 註解盡量放在頁面的右側，並垂直對齊 。
        

## 6. 程式碼基本模板 (Program Template)

這是一個標準的空白程式碼模板（Template.asm），未來撰寫程式可直接套用此架構 ：

程式碼片段

```
.386
.model flat, stdcall
.stack 4096
ExitProcess PROTO, dwExitCode:DWORD

.data
; declare variables here

.code
main PROC
; write your code here

INVOKE ExitProcess,0
main ENDP
END main
```
