# 2.2 32-Bit x86 Processors

## 1. 運作模式 (Modes of Operation)

32位元 x86 處理器支援多種模式，以相容舊有系統並提供現代安全功能：

- **保護模式 (Protected Mode)**：處理器的原生模式，可使用所有指令與功能 。在此模式下，CPU 會防止程式存取其區段（segments）以外的記憶體，確保系統安全 。
    
- **虛擬-8086 模式 (Virtual-8086 Mode)**：在保護模式的環境下，模擬執行 8086 真實模式的電腦 。每個程式都有獨立的 1 MB 位址空間 。
    
- **真實位址模式 (Real-Address Mode)**：模擬早期 Intel 處理器的運作方式，雖然支援直接存取硬體與記憶體，但容易導致作業系統當機 。
    
- **系統管理模式 (System Management Mode)**：主要由電腦製造商實作，提供電源管理與系統安全性功能 。
    

## 2. 基本執行環境 (Basic Execution Environment)

這部分定義了程式執行時可使用的暫存器與位址空間：

- **位址空間 (Address Space)**：
    
    - **保護模式**：支援高達 4 GB 的 32 位元定址空間 。
        
    - **真實/虛擬-8086 模式**：僅支援 1 MB 的 20 位元定址空間 。
        
- **通用暫存器 (General-Purpose Registers)**：
    
    - 共有八個 32 位元暫存器：**EAX、EBX、ECX、EDX、EBP、ESP、ESI、EDI** 。
        
    - 部分暫存器可拆分為 16 位元（如 AX）或 8 位元（如 AH, AL）來使用 。
        
    - **特殊用途**：
        
        - **EAX**：乘法與除法指令的累加器 。
            
        - **ECX**：迴圈計數器 。
            
        - **ESP**：堆疊指標 (Stack Pointer) 。
            
        - **EBP**：堆疊框指標 (Frame Pointer) 。
            
        - **ESI/EDI**：索引暫存器 (Index Registers) 。
            
- **區段暫存器 (Segment Registers)**：16 位元的暫存器，用於定義記憶體區塊，包括 **CS** (程式碼)、**DS** (資料)、**SS** (堆疊)，以及額外的 **ES、FS、GS** 。
    
- **指令指標與旗標**：
    
    - **EIP**：儲存下一條要執行指令的位址 。
        
    - **EFLAGS**：包含多個 1 位元的狀態旗標，反映運算結果（例如：CF 進位、ZF 零旗標、SF 符號旗標等）。
        

## 3. 多媒體與浮點運算暫存器

為了加速科學運算與多媒體處理，IA-32 提供了額外的暫存器：

- **浮點運算單元 (FPU)**：八個 80 位元的資料暫存器 ($ST(0)$ 到 $ST(7)$)，以堆疊方式排列，用於浮點運算 。
    
- **SIMD 暫存器**：
    
    - **MMX**：八個 64 位元暫存器 。
        
    - **XMM**：八個 128 位元暫存器，用於單指令多資料 (SIMD) 運算，適合多媒體與通訊應用 。
        

## 4. 記憶體管理與保護

- **真實模式限制**：早期的真實模式缺乏保護機制，程式可以存取任何記憶體位址，甚至是其他程式的範圍，容易造成安全性疑慮 。
    
- **保護模式優勢**：支援多工執行，每個程式會被分配到受保護的記憶體分割區，彼此互不干擾 。透過多層級 (multi-level) 保護裝置隔離作業系統與應用程式 。
