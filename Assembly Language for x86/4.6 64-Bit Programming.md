# 4.6 64-Bit Programming

## 本節涵蓋主題

|主題|說明|
|---|---|
|`MOV`（64 位元）|立即值與記憶體運算元搬移的上位元清除規則|
|`MOVSXD`|32 位元符號延伸至 64 位元|
|`OFFSET`（64 位元）|產生 64 位元位址|
|`LOOP`（64 位元）|改用 RCX 作為計數器|
|SumArray 64 位元版|完整程式範例|
|加減法（64 位元）|部分暫存器運算的行為差異|
|間接 / 索引運算元|64 位元定址規則|
|指標（64 位元）|指標變數改為 QWORD|

---

## 1. MOV 指令（64 位元）

64 位元模式下，`MOV` 的來源可以是 8、16、32 或 64 位元的立即值或記憶體運算元，但**上位元的清除行為**依來源型別而異。

### 規則整理

|來源型別|對 64 位元暫存器的影響|
|---|---|
|**立即值**（8 / 16 / 32 位元）|未使用的上位元**全部清零**|
|**32 位元記憶體運算元**|上方 32 位元**清零**|
|**8 / 16 位元記憶體運算元**|上位元**不受影響**|

### 立即值範例

```asm
mov rax, 0ffffffffh   ; RAX = 00000000ffffffff（上 32 位元清零）
mov rax, 06666h       ; RAX = 0000000000006666（bits 16-63 清零）
mov rax, 055h         ; RAX = 0000000000000055（bits  8-63 清零）
```

### 32 位元記憶體運算元

將 32 位元記憶體內容搬入 EAX，會連帶清除 RAX 的上 32 位元。

```asm
.data
myDword DWORD 80000000h

.code
mov rax, 0ffffffffffffffffh   ; RAX = ffffffffffffffff
mov eax, myDword              ; RAX = 0000000080000000（上 32 位元歸零）
```

### 8 / 16 位元記憶體運算元

搬入 AL 或 AX，**不影響** RAX 的其餘位元。

```asm
.data
myByte BYTE 55h
myWord WORD 6666h

.code
mov ax, myWord   ; bits 16-63 不受影響
mov al, myByte   ; bits  8-63 不受影響
```

## 2. MOVSXD（符號延伸搬移，32→64）

將 **32 位元**暫存器或記憶體運算元**符號延伸**後搬入 64 位元目的暫存器。

```asm
mov ebx, 0ffffffffh    ; EBX = 0ffffffffh（視為有符號：-1）
movsxd rax, ebx        ; RAX = ffffffffffffffff（-1 的 64 位元表示）
```

> **與 MOVSX 的差異**：`MOVSX` 用於 8/16 位元來源；`MOVSXD` 專用於 32 位元來源延伸至 64 位元目的。

---

## 3. OFFSET（64 位元）

64 位元模式下，`OFFSET` 產生的是 **64 位元位址**，因此必須存放在 **64 位元暫存器或 QWORD 變數**中。

```asm
.data
myArray WORD 10, 20, 30, 40

.code
mov rsi, OFFSET myArray   ; RSI = myArray 的 64 位元位址
```

---

## 4. LOOP（64 位元）

64 位元模式下，`LOOP` 改用 **RCX**（而非 ECX）作為迴圈計數器，其餘邏輯與 32 位元版相同。

```asm
mov rcx, count    ; 設定迴圈次數（64 位元計數器）
L1:
    .             ; 迴圈主體
    loop L1       ; RCX -= 1；若 RCX ≠ 0 跳回 L1
```

---

## 5. 64 位元版 SumArray 完整範例

計算 QWORD 陣列所有元素的總和，是 32 位元版的 64 位元對應程式。

```asm
; Summing an array  (SumArray_64.asm)
ExitProcess PROTO

.data
intarray QWORD 1000000000000h, 2000000000000h
         QWORD 3000000000000h, 4000000000000h

.code
main PROC
    mov rdi, OFFSET intarray    ; RDI <- 陣列起始位址（64 位元）
    mov rcx, LENGTHOF intarray  ; RCX = 4（迴圈次數，改用 RCX）
    mov rax, 0                  ; 清空累加器
L1:
    add rax, [rdi]              ; RAX += 目前元素（QWORD）
    add rdi, TYPE intarray      ; RDI += 8（QWORD = 8 bytes）
    loop L1                     ; RCX -= 1；不等於 0 則跳回 L1

    mov ecx, 0
    call ExitProcess
main ENDP
END
```

### 32 位元 vs 64 位元 SumArray 對照

|項目|32 位元版|64 位元版|
|---|---|---|
|陣列型別|`DWORD`|`QWORD`|
|位址暫存器|`EDI`|`RDI`|
|迴圈計數器|`ECX`|`RCX`|
|累加器|`EAX`|`RAX`|
|步距（TYPE）|4 bytes|8 bytes|

## 6. 加法與減法（64 位元）

`ADD`、`SUB`、`INC`、`DEC` 的行為與 32 位元模式相同，同樣影響 CPU 旗標（Sign、Carry、Zero、Overflow）。

### 關鍵差異：部分暫存器運算

使用**部分暫存器**（如 AX、AL）進行運算時，**其餘位元不受影響**。

#### 加法範例

```asm
mov rax, 0ffffh   ; RAX = 000000000000ffff
mov bx, 1
add ax, bx        ; AX = ffff + 0001 = 0000（進位），RAX 上方位元不變
                  ; RAX = 0000000000000000
```

#### 減法範例

```asm
; 情境一：使用 32 位元暫存器（EAX/EBX）
mov rax, 0        ; RAX = 0000000000000000
mov ebx, 1
sub eax, ebx      ; EAX = 0 - 1 = ffffffff（借位）
                  ; 因為是 32 位元寫入，RAX 上方 32 位元清零
                  ; RAX = 00000000ffffffff

; 情境二：使用 16 位元暫存器（AX/BX）
mov rax, 0        ; RAX = 0000000000000000
mov bx, 1
sub ax, bx        ; AX = 0 - 1 = ffff（借位）
                  ; 上方 48 位元不受影響
                  ; RAX = 000000000000ffff
```

## 7. 間接運算元（64 位元）

64 位元模式下，間接運算元**必須使用 64 位元通用暫存器**（如 RSI、RDI、RBX），並搭配 `PTR` 指定目標大小。

```asm
dec BYTE  PTR [rdi]    ; 對 RDI 所指的 1 byte 做 DEC
inc WORD  PTR [rbx]    ; 對 RBX 所指的 2 bytes 做 INC
inc QWORD PTR [rsi]    ; 對 RSI 所指的 8 bytes 做 INC
```

---

## 8. 索引運算元（64 位元）

64 位元模式的索引定址與 32 位元相同，但 QWORD 陣列的**比例因子（Scale Factor）為 8**。

```asm
.data
array QWORD 1, 2, 3, 4

.code
mov rsi, 3                       ; 邏輯索引（第 4 個元素，從 0 起算）
mov rax, array[rsi * 8]          ; RAX = 4
; 或寫成 array[rsi * TYPE array]，可讀性更高
```

---

## 9. 指標（64 位元）

64 位元模式下，指標變數必須宣告為 **QWORD**（儲存 64 位元偏移量）。

```asm
.data
arrayB BYTE 10h, 20h, 30h, 40h
ptrB   QWORD arrayB           ; 儲存 arrayB 的 64 位元位址

; 等效寫法：
ptrB QWORD OFFSET arrayB
```

---

## 重點整理

|項目|32 位元|64 位元|
|---|---|---|
|暫存器前綴|E（EAX、ECX…）|R（RAX、RCX…）|
|迴圈計數器|ECX|RCX|
|OFFSET 位址大小|32 bits|64 bits|
|指標變數型別|DWORD|QWORD|
|QWORD 比例因子|—|×8|
|32 位元運算元搬入 64 位元暫存器|—|**清除上 32 位元**|
|8 / 16 位元記憶體運算元搬入|—|上位元**不受影響**|
|立即值（任意大小）搬入|—|清除**所有未使用位元**|

> **記憶口訣**：「32 位元記憶體會清上半，8/16 位元記憶體不動上半，立即值不管大小全部清上半。」
