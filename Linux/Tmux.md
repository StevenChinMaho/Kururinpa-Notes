`tmux` (Terminal Multiplexer) 是比 `screen` 更強大的終端機多工神器。它允許您在一個終端機連線中開啟多個會話 (Session)、視窗 (Window) 和窗格 (Pane)，並且支援在 SSH 斷線後讓程式繼續在背景執行。

### 安裝 Tmux
在 Ubuntu 系統上:

```bash
sudo apt update
sudo apt install tmux
```

### 核心概念：前綴鍵 (Prefix Key)

`tmux` 的所有快捷鍵操作都需要先按下**前綴鍵**。預設的前綴鍵是 `Ctrl + b`。 操作邏輯：先按下 `Ctrl + b`，**放開後**，再按相對應的指令鍵。

---

### 1. 工作階段管理 (Session)

工作階段是 `tmux` 的最上層單位，可以在背景持續執行。

**在終端機 (非 tmux 環境) 執行的指令：**

- **建立新工作階段：**
    
    ```bash
    tmux                      # 建立預設編號的 Session
    tmux new -s session_name  # 建立一個名為 session_name 的 Session (推薦)
    ```
    
- **列出所有工作階段：**
    
    ```bash
    tmux ls
    ```
    
- **恢復工作階段 (Attach)：**
    
    ```bash
    tmux a                    # 恢復最近使用的一個 Session
    tmux a -t session_name    # 恢復指定名稱的 Session
    ```
    
- **刪除/強制關閉工作階段：**
    
    ```bash
    tmux kill-session -t session_name
    ```
    

**在 tmux 內部使用的快捷鍵：**

- **暫時離開 (Detach)：** `Ctrl + b`，放開後按 `d`。 (程式會繼續在背景跑，並回到原本的 Shell)
    
- **重新命名當前 Session：** `Ctrl + b`，放開後按 `$`。
    

---

### 2. 視窗管理 (Window)

視窗類似於瀏覽器的「分頁 (Tab)」，一個 Session 內可以有多個 Window。

**快捷鍵 (皆須先按 `Ctrl + b`)：**

- `c` (Create)：建立一個新視窗。
    
- `p` (Previous)：切換到上一個視窗。
    
- `n` (Next)：切換到下一個視窗。
    
- `0`~`9`：直接跳到指定編號的視窗。
    
- `w` (Windows)：開啟清單，列出所有視窗並可使用方向鍵選擇切換。
    
- `,` (逗號)：重新命名目前的視窗 (方便辨識正在執行的任務)。
    
- `&`：關閉目前的視窗 (會要求確認)。
    

---

### 3. 窗格管理 (Pane)

窗格可以將目前的視窗「分割」成多個區塊，讓您同時監控多個畫面。

**快捷鍵 (皆須先按 `Ctrl + b`)：**

- **畫面分割：**
    
    - `%`：垂直分割 (左右各一半)。
        
    - `"` (雙引號)：水平分割 (上下各一半)。
        
- **切換游標位置：**
    
    - 方向鍵 (`↑` `↓` `←` `→`)：移動游標到相鄰的窗格。
        
    - `o`：依序輪流切換到下一個窗格。
        
- **調整與操作：**
    
    - `z` (Zoom)：將目前的窗格**放大到全螢幕**，再按一次可還原回分割狀態 (看 Log 或是寫 Code 時超好用)。
        
    - `Ctrl + 方向鍵`：微調目前窗格的邊界大小。
        
    - `x`：關閉目前的窗格 (會要求確認)。
        
    - `Space` (空白鍵)：切換內建的窗格排版模式。

### 4. 複製模式與歷史捲動 (Copy Mode)

在 `tmux` 中，預設情況下是無法直接用滑鼠滾輪往上滑查看歷史訊息的。要捲動畫面或單純靠鍵盤複製文字，必須進入「複製模式」。

**快捷鍵 (皆須先按 `Ctrl + b`)：**

- `[`：進入複製模式。此時畫面右上角會出現指示器（例如顯示目前的行數），接著您就可以使用方向鍵、`Page Up` / `Page Down` 來上下捲動查看歷史輸出。
    
- **純鍵盤複製文字流程：**
    
    1. 進入複製模式 (`Ctrl + b` 放開後按 `[`)。
        
    2. 使用方向鍵移動游標到想要複製的文字起點。
        
    3. 按下 `Space` (空白鍵) 開始選取模式 (開始反白)。
        
    4. 使用方向鍵移動游標，選取您要複製的文字範圍。
        
    5. 按下 `Enter`，`tmux` 會複製選取的內容，並自動離開複製模式。
        
- `]`：貼上剛剛在 `tmux` 複製模式中複製的內容。
    

_(註：如果熟悉 Vim，可以在設定檔中開啟 vi 模式，讓複製模式的操作邏輯與 Vim 一致。)_

### 5. 其他實用操作與快捷鍵

以下是日常使用中也很常遇到的快捷鍵與指令，適合隨時查閱。

**快捷鍵 (皆須先按 `Ctrl + b`)：**

- `?`：顯示所有的 tmux 快捷鍵清單 (按 `q` 或是 `Esc` 離開)。忘記按鍵時非常實用。
    
- `:` (冒號)：進入 `tmux` 的命令列提示字元模式 (畫面最下方會變成黃底黑字)，可以在這裡直接輸入 tmux 的內部指令。
    
- `t` (Time)：在目前的窗格顯示一個大時鐘 (按任意鍵或 `q` 離開)。
    
- `q`：短暫顯示目前所有窗格的編號。如果在數字消失前按下對應的數字，可以直接跳到該窗格。

### 6. 基本自訂設定檔 (進階推薦)

`tmux` 支援高度客製化，所有的設定都可以寫在家目錄下的 `~/.tmux.conf` 檔案中。

**強烈推薦新手加入的設定：開啟滑鼠支援** 預設的純鍵盤操作可能需要時間適應，如果您希望可以用滑鼠點擊切換視窗、拖拉調整窗格大小，或是直接用滾輪捲動歷史訊息，請在終端機執行以下指令建立設定：

```bash
# 將開啟滑鼠支援的指令寫入設定檔
echo "set -g mouse on" >> ~/.tmux.conf
```

**重新載入設定檔 (Reload Config)：** 每次修改 `~/.tmux.conf` 後，必須重新載入才會生效。

- **在 tmux 外部 (一般終端機)：**
    
    ```bash
    tmux source-file ~/.tmux.conf
    ```
    
- **在 tmux 內部：** 按下 `Ctrl + b`，接著按 `:`，然後輸入 `source-file ~/.tmux.conf` 並按 Enter。
